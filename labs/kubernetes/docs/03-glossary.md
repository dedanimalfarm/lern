# Глоссарий (Glossary)

Здесь собраны ключевые термины и концепции, с которыми вы сталкиваетесь в лаборатории.

- **API group/version**: Адрес ресурса в Kubernetes API в виде `группа/версия` + `Kind`. Core-группа пустая (`Pod`/`Service` → `apiVersion: v1`), именованные группы — `apps/v1` (Deployment), `batch/v1` (Job), `networking.k8s.io/v1` (Ingress). Список — `kubectl api-resources`.
- **ApplicationSet**: Ресурс Argo CD, который позволяет декларативно создавать множество `Application` на основе генераторов (например, для разворачивания одного приложения в разных окружениях).
- **Cardinality (Кардинальность)**: В контексте метрик и логов (Prometheus/Loki) — количество уникальных комбинаций лейблов (labels). Высокая кардинальность (например, лейбл `user_id` с миллионами значений) сильно нагружает базу данных.
- **CNI (Container Network Interface)**: Плагин, который выдаёт подам IP-адреса и обеспечивает сеть между ними (в нашем кластере — Calico). От CNI зависит, реально ли работают `NetworkPolicy` (Calico/Cilium — да; некоторые — игнорируют молча).
- **Control-plane (управляющий слой)**: «Мозг» кластера: `kube-apiserver` (единая точка входа в API), `etcd` (хранит всё состояние), `kube-scheduler` (выбирает ноду поду), `kube-controller-manager` (контроллеры reconcile). Запускается на control-plane-нодах.
- **CSI (Container Storage Interface)**: Стандартный интерфейс, позволяющий Kubernetes работать с различными системами хранения данных (AWS EBS, Ceph, local-path) через единый API.
- **Failover / Switchover**: Переключение трафика с отказавшего узла (Primary) на резервный (Replica). Failover — автоматическое переключение при сбое. Switchover — запланированное ручное переключение.
- **Kustomize**: Встроенный в `kubectl` инструмент для кастомизации Kubernetes манифестов без шаблонизации. Позволяет создавать базовые манифесты (base) и накладывать на них патчи (overlays) для разных окружений.
- **NodeSelector / Affinity**: Механизмы Kubernetes для указания того, на каких узлах должен запускаться под (например, только на узлах с дисками SSD).
- **Operator**: Метод упаковки, развёртывания и управления Kubernetes-приложением. Оператор использует CRD (Custom Resource Definition) и контроллер для автоматизации задач, которые обычно выполняет инженер (например, создание бэкапов БД).
- **PSA (Pod Security Admission)**: Встроенный в Kubernetes механизм ограничения прав подов (Privileged, Baseline, Restricted). Заменил устаревший PodSecurityPolicy (PSP).
- **QoS (Quality of Service)**: Класс обслуживания пода (Guaranteed, Burstable, BestEffort), который Kubernetes назначает на основе requests и limits. Влияет на то, какие поды будут убиты первыми при нехватке памяти (OOM).
- **RBAC (Role-Based Access Control)**: Управление правами в кластере. `Role`/`ClusterRole` описывают права (`apiGroups × resources × verbs`), `RoleBinding`/`ClusterRoleBinding` привязывают их к субъекту (ServiceAccount, пользователю, группе). Проверить право: `kubectl auth can-i`.
- **Reconcile / Reconciliation loop (Реконсиляция)**: Основной цикл работы контроллеров Kubernetes. Контроллер постоянно сравнивает *текущее состояние* (фактическое) с *желаемым состоянием* (описанным в манифестах) и предпринимает действия для их синхронизации.
- **Sync-wave**: В Argo CD — фаза синхронизации ресурсов. Ресурсы с меньшим номером (например, `-1`) применяются раньше ресурсов с бóльшим номером (например, `1`).
- **Taint / Toleration**: Механизм, позволяющий узлу "отталкивать" поды (taint), если у тех нет соответствующего "допуска" (toleration).
- **VAP (ValidatingAdmissionPolicy)**: Механизм Kubernetes (на базе языка CEL) для валидации ресурсов при их создании или изменении. Позволяет писать политики безопасности без использования внешних webhook-ов (вроде OPA Gatekeeper или Kyverno).
