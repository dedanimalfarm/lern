# Глоссарий (Glossary)

Здесь собраны ключевые термины и концепции, с которыми вы сталкиваетесь в лаборатории.

- **Admission Controller (Контроллер допуска)**: Компонент API-сервера, перехватывающий запросы после аутентификации/авторизации, но до их сохранения в etcd. Бывают Validating (только проверяют) и Mutating (могут изменять запрос).
- **Affinity / Anti-Affinity (Аффинность / Анти-аффинность)**: Правила, определяющие, на какие узлы следует или не следует планировать под (Node Affinity) или рядом с какими другими подами он должен/не должен работать (Pod Affinity/Anti-Affinity).
- **Annotation (Аннотация)**: Метаданные (ключ-значение), прикрепляемые к объектам для хранения произвольной неидентифицирующей информации (в отличие от Labels).
- **API group/version (Группа/Версия API)**: Адрес ресурса в Kubernetes API в виде `группа/версия` + `Kind`. Core-группа пустая (`Pod`/`Service` → `apiVersion: v1`), именованные группы — `apps/v1` (Deployment), `batch/v1` (Job). Список — `kubectl api-resources`.
- **ApplicationSet**: Ресурс Argo CD, который позволяет декларативно создавать множество `Application` на основе генераторов (например, для разворачивания одного приложения в разных окружениях).
- **Argo CD**: Декларативный инструмент непрерывной доставки (Continuous Delivery) для Kubernetes, реализующий подход GitOps.
- **Blue/Green Deployment (Сине-зеленое развертывание)**: Стратегия деплоя, при которой создается полностью новая (Green) версия окружения рядом со старой (Blue), после чего трафик переключается целиком на новую.
- **Canary Deployment (Канареечное развертывание)**: Стратегия деплоя, при которой новая версия приложения постепенно получает часть трафика (например, 5%), что позволяет проверить ее на реальных пользователях перед полным переключением.
- **Cardinality (Кардинальность)**: В контексте метрик и логов (Prometheus/Loki) — количество уникальных комбинаций лейблов (labels). Высокая кардинальность (например, лейбл `user_id` с миллионами значений) сильно нагружает базу данных.
- **CEL (Common Expression Language)**: Легковесный язык выражений от Google, используемый в Kubernetes для ValidatingAdmissionPolicy (VAP) и CRD валидации.
- **CNI (Container Network Interface)**: Плагин, который выдаёт подам IP-адреса и обеспечивает сеть между ними (в нашем кластере — Calico). От CNI зависит, реально ли работают `NetworkPolicy` (Calico/Cilium — да; некоторые — игнорируют молча).
- **ConfigMap**: Объект API для хранения неконфиденциальных данных в виде пар ключ-значение. Обычно монтируется в поды как файлы или переменные окружения.
- **Control-plane (Управляющий слой)**: «Мозг» кластера: `kube-apiserver` (единая точка входа в API), `etcd` (хранит всё состояние), `kube-scheduler` (выбирает ноду поду), `kube-controller-manager` (контроллеры reconcile). Запускается на control-plane-нодах.
- **Controller (Контроллер)**: Фоновый процесс, который следит за состоянием кластера через API-сервер и приводит текущее состояние к желаемому (описанному в ресурсах).
- **CRD (Custom Resource Definition)**: Механизм расширения API Kubernetes, позволяющий добавлять собственные типы ресурсов (Kinds), с которыми можно работать так же, как со стандартными подами или сервисами.
- **CronJob**: Ресурс, который управляет заданиями (Jobs), запускаемыми по расписанию (в формате cron).
- **CSI (Container Storage Interface)**: Стандартный интерфейс, позволяющий Kubernetes работать с различными системами хранения данных (AWS EBS, Ceph, local-path) через единый API.
- **DaemonSet**: Ресурс, гарантирующий, что определенный под будет запущен на всех (или на выбранных по меткам) узлах кластера (например, агенты логирования, CNI).
- **Deployment (Развертывание)**: Ресурс верхнего уровня, который управляет ReplicaSet-ами и обеспечивает декларативное обновление подов (RollingUpdate).
- **DRA (Dynamic Resource Allocation / Динамическое выделение ресурсов)**: API (alpha в 1.26+) для выделения и управления специфическими ресурсами (например, GPU) более гибко, чем стандартные request/limit.
- **EndpointSlice / Endpoints**: Ресурсы, которые хранят сетевые адреса (IP) подов, на которые сервис должен маршрутизировать трафик. EndpointSlice — более новая и масштабируемая версия Endpoints.
- **Etcd**: Распределенное, согласованное key-value хранилище, используемое Kubernetes в качестве единого источника истины для всех данных кластера.
- **Eviction (Вытеснение)**: Процесс принудительного завершения пода узлом (kubelet), например, при нехватке ресурсов (Memory Pressure, Disk Pressure).
- **Failover / Switchover**: Переключение трафика с отказавшего узла (Primary) на резервный (Replica). Failover — автоматическое переключение при сбое. Switchover — запланированное ручное переключение.
- **Field Selector (Селектор полей)**: Механизм фильтрации ресурсов при запросе к API-серверу на основе значений полей ресурса (например, `kubectl get pods --field-selector status.phase=Running`).
- **Finalizer (Финализатор)**: Ключ в метаданных ресурса, который сообщает Kubernetes, что перед полным удалением объекта из etcd необходимо выполнить определенные очищающие действия. Пока финализатор не удален контроллером, объект висит в состоянии `Terminating`.
- **Gateway API**: Современный, стандартизированный API для управления маршрутизацией трафика (L4/L7) в Kubernetes, призванный заменить устаревающий Ingress. Использует ресурсы GatewayClass, Gateway, HTTPRoute.
- **GitOps**: Парадигма, при которой Git-репозиторий является единственным источником истины для декларативной инфраструктуры и приложений. Изменения автоматически применяются агентом (например, Argo CD).
- **Headless Service (Service без ClusterIP)**: Сервис, у которого `clusterIP: None`. Он не балансирует трафик, а напрямую отдает DNS-ответ с IP-адресами всех подов, соответствующих селектору (часто используется со StatefulSet).
- **Helm**: Пакетный менеджер для Kubernetes, использующий шаблоны (Charts) для параметризации и развертывания сложных приложений.
- **HPA (Horizontal Pod Autoscaler)**: Ресурс, автоматически изменяющий количество реплик (подов) в Deployment/StatefulSet на основе наблюдаемых метрик (например, загрузка CPU).
- **Informer (Информер)**: Компонент в клиентских библиотеках (client-go), который поддерживает локальный кэш объектов из API-сервера и реагирует на их изменения (Add/Update/Delete) без необходимости постоянного поллинга.
- **Ingress**: Объект API, управляющий внешним доступом к сервисам в кластере (обычно HTTP/HTTPS), может обеспечивать балансировку, SSL-терминацию и name-based virtual hosting. Заменяется на Gateway API.
- **Init Container (Init-контейнер)**: Контейнер, который запускается и должен успешно завершиться до старта основных контейнеров пода (например, для выполнения миграций БД или подготовки конфигурации).
- **Job (Задание)**: Ресурс, который создает один или несколько подов и гарантирует, что заданное их количество успешно завершится.
- **JobSet**: Ресурс (в составе экосистемы Kubernetes-sigs), управляющий группой связанных Jobs как единым целым (полезно для распределенного машинного обучения и batch-обработки).
- **Karpenter / Cluster Autoscaler**: Инструменты автомасштабирования кластера. CA добавляет готовые узлы (NodeGroups), тогда как Karpenter (более современный) динамически provision-ит оптимальные узлы "на лету" под требования подов.
- **Kube-proxy**: Сетевой прокси на каждом узле, реализующий концепцию Services (маршрутизирует трафик к IP-адресам подов через iptables/IPVS).
- **Kubelet**: Главный "агент" (сервис), работающий на каждом узле. Регистрирует узел в apiserver, следит за тем, чтобы нужные контейнеры были запущены в подах и были здоровы.
- **Kustomize**: Встроенный в `kubectl` инструмент для кастомизации Kubernetes манифестов без шаблонизации. Позволяет создавать базовые манифесты (base) и накладывать на них патчи (overlays) для разных окружений.
- **Label (Метка)**: Пары ключ-значение, прикрепляемые к объектам для идентификации и группировки. Используются селекторами (Label Selectors) для выбора ресурсов (например, сервисом для выбора подов).
- **LimitRange**: Ресурс, задающий ограничения ресурсов по умолчанию (requests/limits) для подов или контейнеров в определенном namespace.
- **Liveness Probe (Проба живучести)**: Проверка, позволяющая kubelet узнать, жив ли контейнер. Если провалена, kubelet перезапускает контейнер.
- **LogQL**: Язык запросов к логам в Grafana Loki (похожий на PromQL), позволяющий фильтровать логи и извлекать из них метрики.
- **Manifest (Манифест)**: YAML или JSON файл, описывающий желаемое состояние объекта Kubernetes (Deployment, Service и т.д.).
- **Metric Server**: Легковесный компонент, собирающий базовые метрики использования ресурсов (CPU/RAM) с kubelet-ов для обеспечения работы HPA и `kubectl top`.
- **Namespace (Пространство имен)**: Механизм логического разделения кластера на виртуальные подкластеры. Ресурсы в одном namespace должны иметь уникальные имена.
- **NetworkPolicy (Сетевая политика)**: Ресурс, описывающий правила L3/L4/L7 (в зависимости от CNI) разрешающие или запрещающие сетевой трафик между подами и внешним миром.
- **Node (Узел/Нода)**: Рабочая машина в Kubernetes (виртуальная или физическая), на которой запускаются поды.
- **NodeSelector / Affinity**: Механизмы Kubernetes для указания того, на каких узлах должен запускаться под (например, только на узлах с дисками SSD).
- **OOMKilled (Out Of Memory)**: Состояние завершения контейнера, которое возникает, когда процесс внутри превышает выделенный ему лимит памяти (memory limit) и убивается ядром Linux.
- **Operator (Оператор)**: Паттерн упаковки, развёртывания и управления Kubernetes-приложением. Оператор использует CRD (Custom Resource Definition) и контроллер для автоматизации задач (например, бэкапов БД).
- **OwnerReference**: Поле в метаданных дочерних объектов (например, ReplicaSet), указывающее на родительский объект (например, Deployment). Используется для каскадного удаления (Garbage Collection).
- **PDB (Pod Disruption Budget)**: Ресурс, ограничивающий количество подов приложения, которые могут быть временно недоступны из-за добровольных прерываний (evictions, drain узла).
- **Pod (Под)**: Минимальная развертываемая единица в Kubernetes, представляющая один или несколько контейнеров, делящих общую сеть (IP), хранилище и жизненный цикл.
- **PromQL**: Мощный язык запросов Prometheus, позволяющий агрегировать метрики временных рядов (time-series).
- **PSA (Pod Security Admission)**: Встроенный в Kubernetes механизм ограничения прав подов (Privileged, Baseline, Restricted). Заменил устаревший PodSecurityPolicy (PSP).
- **PVC / PV (PersistentVolumeClaim / PersistentVolume)**: Механизм абстрагирования хранилища. PV — физический том в кластере, PVC — запрос пользователя на хранилище с определенным размером и классом (StorageClass).
- **QoS (Quality of Service)**: Класс обслуживания пода (Guaranteed, Burstable, BestEffort), который назначается на основе requests и limits. Влияет на то, какие поды будут вытеснены (OOM-killed/evicted) первыми.
- **RBAC (Role-Based Access Control)**: Управление правами в кластере. `Role`/`ClusterRole` описывают права, `RoleBinding`/`ClusterRoleBinding` привязывают их к субъекту (ServiceAccount, user). Проверить: `kubectl auth can-i`.
- **Readiness Probe (Проба готовности)**: Проверка, позволяющая kubelet узнать, готов ли контейнер принимать трафик. Если провалена, IP пода удаляется из EndpointSlice сервиса.
- **Reconcile / Reconciliation loop (Реконсиляция)**: Основной цикл работы контроллеров Kubernetes. Постоянно сравнивает фактическое состояние с желаемым (описанным в манифестах) и синхронизирует их.
- **ReplicaSet**: Ресурс, гарантирующий, что определенное количество реплик пода всегда запущено. Обычно управляется через Deployment, а не напрямую.
- **Resource Quota (Ресурсная квота)**: Ресурс, ограничивающий общее потребление ресурсов (CPU, память, количество PV/Service) в рамках определенного namespace.
- **Requests / Limits (Запросы и Лимиты)**: Параметры ресурсов контейнера. Request — гарантированный минимум, необходимый для старта (используется планировщиком). Limit — жесткий максимум потребления (приводит к throttle CPU или OOMKilled RAM).
- **Role / ClusterRole**: В RBAC это наборы разрешений на ресурсы (`apiGroups × resources × verbs`). Role — в рамках namespace, ClusterRole — в рамках всего кластера.
- **Rolling Update**: Стратегия обновления (по умолчанию в Deployment), при которой поды новой версии поднимаются постепенно, а старой удаляются, обеспечивая отсутствие downtime.
- **Secret (Секрет)**: Ресурс для хранения конфиденциальной информации (пароли, токены, ключи TLS). Данные хранятся в формате base64 (по умолчанию не зашифрованы в etcd, если не настроен KMS).
- **Service (Сервис)**: Абстракция, определяющая логический набор подов (через Label Selector) и политику доступа к ним (предоставляя стабильный ClusterIP).
- **ServiceAccount**: Учетная запись, используемая процессами (подами), работающими в кластере, для аутентификации в API-сервере Kubernetes.
- **Sidecar container (Сайдкар контейнер)**: Паттерн вспомогательного контейнера в поде (например, прокси, агент логов). С v1.29 поддерживаются нативно как init-контейнеры с `restartPolicy: Always`.
- **Startup Probe (Стартовая проба)**: Проверка для медленно стартующих приложений. Пока она не будет успешной, liveness и readiness пробы не запускаются.
- **StatefulSet**: Ресурс для управления stateful-приложениями (базы данных, очереди). Гарантирует упорядоченный деплой, уникальные стабильные сетевые имена (pod-0, pod-1) и стабильные PVC.
- **StorageClass**: Класс хранилища, описывающий провайдера (provisioner) и параметры для динамического создания PersistentVolumes.
- **Sync-wave**: В Argo CD — фаза синхронизации ресурсов. Ресурсы с меньшим номером применяются раньше ресурсов с бóльшим номером.
- **Taint / Toleration**: Механизм, позволяющий узлу "отталкивать" (taint) поды, если у тех нет соответствующего "допуска" (toleration). Полезно для выделения узлов под специфичные нагрузки (GPU).
- **VAP (ValidatingAdmissionPolicy)**: Механизм валидации ресурсов (GA с 1.30) при их создании/изменении на базе языка CEL без необходимости разворачивать внешние webhook (OPA/Kyverno).
- **VPA (Vertical Pod Autoscaler)**: Механизм, который анализирует историческое потребление ресурсов подами и автоматически рекомендует или применяет оптимальные значения CPU/RAM Requests & Limits.
- **Webhook (Mutating / Validating Admission Webhook)**: Внешний HTTP-сервис, вызываемый API-сервером во время создания/изменения объекта. Mutating может менять объект (например, вставлять sidecar), Validating — отклонить запрос.
