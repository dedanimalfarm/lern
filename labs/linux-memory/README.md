# Linux Memory Management Labs — Практическое управление памятью в Linux

Этот мини-курс посвящен изучению подсистемы виртуальной памяти Linux: от базовых принципов работы трансляции адресов (MMU) и механизмов выделения памяти (`malloc`/`mmap`) до ограничений ресурсов cgroups v2, тюнинга параметров ядра (THP, Swap, swappiness) и глубокого поиска утечек и OOM-инцидентов.

---

## 🗺 Карта лабораторных работ

Мини-курс состоит из **3 практических работ**, расположенных по нарастающей сложности:

| № | Директория | Тема лабораторной работы | Сложность | Время | Ключевые технологии |
|:-:|---|---|:-:|:-:|---|
| **01** | [`lab01/`](./lab01/) | Управление памятью в Linux: Основы и Инциденты | Средне | 75 мин | `free`, `vmstat`, `pmap`, `swappiness`, `SysV shm`, OOM |
| **02** | [`lab02/`](./lab02/) | Ограничение ресурсов cgroup v2 (Memory Controller) | Средне | 45 мин | `cgroups v2`, `memory.max`, `memory.high`, `oom_score_adj` |
| **03** | [`lab03/`](./lab03/) | Transparent Hugepages (THP) и СУБД | Средне | 45 мин | `Hugepages`, `AnonHugePages`, `CoW write-amplification` |

## 📈 Связь с другими курсами

Понимание принципов управления памятью является фундаментом для более сложных системных и контейнерных технологий:
- **`labs/linux-cgroups`** — детальное изучение ограничений CPU и I/O ресурсов.
- **`labs/kubernetes`** — управление памятью подов через requests/limits (под капотом работают cgroup `memory.max` и `memory.high`) и вытеснение подов на основе OOM QoS-классов (`oom_score_adj`).
- **`labs/linux-troubleshooting`** — диагностика инцидентов с памятью в реальных сценариях (утечки памяти, OOM).

## 💻 Требования к окружению

1. **Операционная система**: Любой современный дистрибутив Linux (рекомендуется Ubuntu 20.04+ / Debian 11+).
2. **Версия ядра**: 4.15+ (для полноценной поддержки cgroup v2 и контроллера памяти).
3. **Права доступа**: Требуются права суперпользователя (`root`/`sudo`) для создания cgroups, настройки swap-файлов и изменения флагов ядра (Hugepages).
4. **Установленные пакеты**:
   ```bash
   sudo apt-get update && sudo apt-get install -y \
     stress-ng procps util-linux python3 sysstat
   ```

## 🧹 Общие правила уборки

Виртуальные лимиты и файлы ядра создаются в оперативной памяти и изменяют текущую конфигурацию системы. Чтобы вернуть систему в исходное состояние, в каждой лабораторной предусмотрен скрипт автоматического сброса:
```bash
# Выполните в папке конкретной лабы
bash cleanup.sh
```
Рекомендуется выполнять очистку ресурсов по окончании каждой практической работы.
