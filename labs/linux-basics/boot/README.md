## Введение
Загрузка Linux-сервера в режиме UEFI (Unified Extensible Firmware Interface) — это многоэтапный процесс, включающий в себя инициализацию аппаратного обеспечения, поиск загрузочных приложений, запуск ядра и инициализацию пользовательских сервисов.

## Теория: Этапы загрузки

### 1. Инициализация прошивки UEFI
1. **Power-On Self-Test (POST):**  
   После включения питания прошивка проверяет процессор, ОЗУ и базовые устройства.
2. **Инициализация устройств UEFI:**  
   Загружаются драйверы прошивки для контроллеров дисков и сети. На этом этапе формируется среда для работы UEFI-приложений.

### 2. Поиск и запуск загрузчика
1. **EFI System Partition (ESP):**  
   Специальный раздел (обычно `/boot/efi`), отформатированный в FAT32, содержащий исполняемые файлы `.efi`.
2. **Выбор и запуск загрузчика:**  
   По переменным UEFI (Boot Variables) находится и запускается загрузчик (например, `\EFI\redhat\grubx64.efi` или `\EFI\systemd\systemd-bootx64.efi`).

### 3. Действия загрузчика (GRUB2, systemd-boot и т.д.)
Загрузчик предлагает меню выбора ядра, загружает образ ядра (`vmlinuz`) и `initrd` (временную файловую систему), затем передаёт управление ядру, передавая ему необходимые параметры.

### 4. Инициализация ядра
Ядро распаковывается в ОЗУ, инициализирует оборудование и монтирует временный корень из `initrd`. После загрузки нужных драйверов монтируется основная файловая система, затем запускается процесс `init` (сегодня это `systemd`).

### 5. Запуск пользовательских сервисов
`systemd` анализирует конфигурацию (`target`-ы) и параллельно запускает системные службы (сеть, логирование, SSH, базы данных). После запуска всех требуемых юнитов система готова к работе.

## Практика: Изучение конфигурации загрузки

В этом задании мы исследуем текущую конфигурацию загрузки вашей системы.

1. **Проверьте, загружена ли система в режиме UEFI:**
   ```bash
   ls /sys/firmware/efi
   ```
   Если директория существует, система загружена в режиме UEFI.

2. **Посмотрите, какой systemd target используется по умолчанию:**
   ```bash
   systemctl get-default
   ```
   Это покажет, в какой режим загружается система (например, `multi-user.target` для консольной работы без GUI или `graphical.target` для GUI).

3. **Задание для проверки:**
   Сохраните вывод команды `systemctl get-default` в файл `boot_info.txt` в вашей домашней директории:
   ```bash
   systemctl get-default > ~/boot_info.txt
   ```

## Troubleshooting — частые проблемы

1. **Система не видит загрузочный диск (Missing OS / No bootable device)**
   - *Причина:* Поврежден EFI System Partition (ESP) или слетели Boot-переменные UEFI в NVRAM.
   - *Решение:* Загрузиться с Live USB, переустановить загрузчик (например, `grub-install`) и обновить конфигурацию загрузчика.

2. **Kernel Panic после выбора ОС в GRUB**
   - *Причина:* Поврежден файл `initrd` (initramfs) или ядро не может найти и смонтировать корневую файловую систему из-за неверного UUID в `fstab`.
   - *Решение:* Загрузиться со старым ядром (выбрать предыдущую версию в меню загрузчика). Если помогло, пересобрать `initramfs` (утилитами `dracut` или `update-initramfs`).

3. **Зависание на этапе загрузки systemd (A start job is running...)**
   - *Причина:* Какой-то критичный сервис или монтирование диска из `/etc/fstab` недоступно (например, сетевой диск без опции `nofail`).
   - *Решение:* Загрузиться в emergency mode (добавив в GRUB параметр `systemd.unit=emergency.target`), затем исправить `fstab` или отключить зависающий сервис.

## Контрольные вопросы

1. В чём ключевое отличие раздела EFI (ESP) от обычного `/boot` раздела в старых BIOS-системах? Какая файловая система на нём используется и почему?
2. Что такое `initrd` (или `initramfs`) и почему без него современные Linux-дистрибутивы обычно не могут смонтировать корневую файловую систему?
3. Какой параметр в конфигурации загрузчика (GRUB) нужно добавить к строке запуска ядра, чтобы система загрузилась в однопользовательском (rescue) режиме для восстановления?
4. За что отвечают UEFI Boot Variables и какой консольной утилитой в Linux можно их посмотреть или изменить?

## Уборка

После успешного прохождения модуля и ответов на вопросы, уберите за собой созданные файлы с помощью скрипта очистки:

./cleanup.sh
