# Lab 11: Docker under the hood (Контейнерная сеть своими руками)

В этой лабе мы напишем bash-скрипт `mini-docker.sh`, который повторяет то, что делает Docker при запуске контейнера с пробросом портов (например, `docker run -p 8080:80`).

Никакой магии, только Linux Bridge, Network Namespaces, veth-пары и iptables NAT!

## Использование "Мини-Докера"

### 1. Запуск контейнера с пробросом порта
Скрипт `mini-docker.sh` принимает имя контейнера и порты (HOST:CONTAINER). Внутри "контейнера" (неймспейса) мы поднимем простейший python web-сервер на указанном внутреннем порту.

```bash
cd /root/lern/labs/linux-basics/networking/lab11_mini_docker
bash mini-docker.sh run myweb 8080:80
```
*Эта команда:*
1. Создаст мост `docker-br0` (аналог `docker0`).
2. Создаст неймспейс `myweb`.
3. Подключит его к мосту и выдаст IP (например, `10.11.0.2`).
4. Настроит NAT (SNAT для выхода в интернет, DNAT для проброса `8080 -> 10.11.0.2:80`).
5. Запустит фоновый HTTP-сервер.

### 2. Проверка
Теперь вы можете обратиться к порту `8080` на вашем хосте (как в настоящем докере)!
```bash
curl http://localhost:8080
```

### 3. Остановка и очистка
```bash
bash mini-docker.sh stop myweb
```

## Как это работает?
Откройте скрипт `mini-docker.sh` в текстовом редакторе. Вы увидите, что это ровно те же команды, которые мы использовали в Lab 6 (Linux Bridge) и Lab 7 (NAT). Единственное отличие — автоматический подсчет IP-адресов и правило проброса портов `iptables -t nat -A PREROUTING`!
