# Lab 10: Динамическая маршрутизация (BGP)

В этой лабе мы развернем 3 виртуальных маршрутизатора (r1, r2, r3), соединенных "цепочкой" `r1 <---> r2 <---> r3`. Мы настроим протокол **BGP (Border Gateway Protocol)** — главный протокол интернета, используя легковесный демон **BIRD**.

## Запуск стенда
Скрипт автоматически создаст неймспейсы, линки и сгенерирует конфигурации для демона маршрутизации.
```bash
cd /root/lern/labs/linux-basics/networking/lab10_bgp
bash setup.sh
```

## Как устроена топология:
- `r1` (AS 65001). Сеть за роутером: `10.1.0.0/24` (интерфейс `dummy0`)
- `r2` (AS 65002). Транзитный роутер.
- `r3` (AS 65003). Сеть за роутером: `10.3.0.0/24` (интерфейс `dummy0`)

По умолчанию `r1` ничего не знает о сети `10.3.0.0/24` (она на `r3`), а `r3` ничего не знает о `10.1.0.0/24`. Но благодаря BGP они обменяются маршрутами через транзитный узел `r2`!

## Задание 1: Проверка таблиц маршрутизации ядра

Посмотрим, как ядро ОС на `r1` видит маршруты.
```bash
ip netns exec r1 ip route
```
Вы увидите, что помимо directly connected маршрутов появился путь к сети `10.3.0.0/24` через `r2`! 
Маршрут установлен демоном маршрутизации (в выводе будет указано `proto bird`).

Проверим связность (пинг от `r1` до виртуальной сети на `r3`):
```bash
ip netns exec r1 ping -c 3 10.3.0.1
```

## Задание 2: Изучение состояния BGP

Подключимся к консоли управления демоном маршрутизации BIRD на `r1`, чтобы посмотреть состояние BGP-сессий:
```bash
ip netns exec r1 birdc -s /tmp/bird-r1.ctl
```

Внутри консоли `bird>` введите команды:
1. `show protocols` (вы увидите статус BGP сессии с `r2` — должно быть состояние `Established`)
2. `show route` (вы увидите все полученные и отправляемые маршруты)
3. `exit` (для выхода)

С помощью BGP (и OSPF) работают все крупнейшие дата-центры и облачные провайдеры!
