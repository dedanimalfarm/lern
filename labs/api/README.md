# Курс: Работа с API — HTTP, JSON, Postman, интеграции

Практический курс для подготовки ко **второй линии поддержки (L2)**.
Закрывает типовое требование вакансий:

> **«Работа с API и инструментами тестирования (Postman, JSON).
> Понимание принципов интеграции систем. Нужно уметь читать техническую
> документацию, понимать формат данных JSON и уметь отправлять/проверять
> POST/GET запросы».**

Формат — как в [`labs/kubernetes/modules`](../kubernetes/modules/): теория
перед каждой частью, команды с реальными ожидаемыми выводами, контрольные
вопросы, broken-сценарии с решениями и автопроверка `verify/verify.sh`.

## Учебный стенд

Модули 01–06 работают на **локальном Helpdesk API** — учебной тикет-системе
на чистой стандартной библиотеке Python (без pip, без Docker):

```bash
# Поднять стенд (порт 8080)
scripts/api.sh up

# Проверить
curl -s http://localhost:8080/health
# {"status":"ok","uptime_s":0}

# Документация API: docs/openapi.yaml (она же на http://localhost:8080/openapi.yaml)

# Погасить
scripts/api.sh down
```

Стенд умеет то, что нужно для отработки реальных навыков L2: три режима
аутентификации, rate limit, вебхуки, идемпотентность и **управляемые
поломки** (таймауты, 500-ки, битый JSON, неверный Content-Type) — на них
построен модуль 06. Сервер: `common/server/helpdesk_api.py`.

Модуль 07 — единственный, которому нужны интернет и бесплатный аккаунт
Atlassian: разворачивается настоящий Jira Service Management (cloud free).

### Требования

- Linux/WSL2 с `python3` (3.9+), `curl`, `jq`;
- Postman на хосте (для модуля 04; из Windows к WSL2 — просто `http://localhost:8080`);
- для модуля 07 — браузер и почта для регистрации Atlassian free tier.

## Как проходить модуль

1. Прочитать «Теорию для изучения перед частью» — затем выполнить команды части.
2. Сверять свой вывод с «ожидаемым» в README (выводы сняты с живого стенда).
3. Ответить на контрольные вопросы (вслух, как на собеседовании).
4. Сделать `tasks/*.md` и разобрать `broken/scenario-*` (решения — в `solutions/`).
5. Запустить `bash verify/verify.sh` — до строки `[OK] module NN verified`.

## Чек-лист навыков (он же — план ответа на собеседовании)

После курса вы должны уверенно, с примерами из лаб:

- [ ] объяснить путь HTTP-запроса: метод → URL → заголовки → тело → статус-код ответа;
- [ ] назвать назначение статус-кодов 200/201/204/301/400/401/403/404/405/415/422/429/500/502/504;
- [ ] прочитать и написать JSON руками; найти ошибку в битом JSON;
- [ ] вытащить нужное поле из ответа API: `jq '.items[].title'`;
- [ ] отправить GET/POST/PATCH/DELETE из curl и из Postman;
- [ ] собрать Postman-коллекцию с environment и авто-тестами;
- [ ] по документации незнакомого API самостоятельно составить корректный запрос;
- [ ] объяснить разницу 401 vs 403, PUT vs PATCH, API key vs Bearer-токен;
- [ ] разобрать инцидент «интеграция не работает»: таймаут / 5xx / битый ответ / не тот Content-Type;
- [ ] объяснить, зачем нужны идемпотентность и ретраи, что такое вебхук;
- [ ] грамотно эскалировать в L3: что воспроизвёл, что проверил, запрос/ответ, request-id.

## Типовые вопросы собеседования L2 (закрываются курсом)

1. Чем GET отличается от POST? Почему GET-запросы нельзя использовать для изменения данных? *(м01)*
2. Пользователь говорит «не работает». Какие статус-коды вы ожидаете увидеть в логах и что каждый означает? *(м01, м06)*
3. Что такое JSON? Какие типы данных в нём есть? Чем отличается `null` от пустой строки? *(м02)*
4. Как проверить, что API живой, если фронтенд не открывается? *(м01, м06)*
5. В Postman запрос работает, а из приложения — нет. Ваши действия? *(м04, м05, м06)*
6. API возвращает 401. API возвращает 403. В чём разница и куда смотреть? *(м05)*
7. Интеграция дублирует заявки. Почему это бывает и как лечится? *(м06 — ретраи + идемпотентность)*
8. Что такое вебхук и чем он отличается от опроса API (polling)? *(м06)*
9. Как создать задачу в Jira через API? *(м07)*
10. Перед вами документация незнакомого API. С чего начнёте? *(м03)*
