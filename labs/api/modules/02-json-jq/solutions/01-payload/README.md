# Решение: payload.json

Три ошибки и почему так:

1. **`// комментарий`** — в стандарте JSON (RFC 8259) комментариев нет.
   Редакторы, «прощающие» их, работают с JSONC — другим форматом.
2. **`'Импорт из CRM…'`** — строки только в двойных кавычках. Одинарные —
   синтаксис Python/JS, откуда их обычно и копируют.
3. **Висячая запятая** после `"requester": …` — в JSON запятая разделяет
   элементы, а не завершает их (в отличие от Python, где trailing comma
   легальна, — вторая по частоте «питоновская» ошибка в JSON).

Диагностический паттерн: валидатор указывает line/column ПЕРВОЙ ошибки;
правим её — запускаем снова, пока не станет валидным. Не пытайтесь найти
все ошибки за один проход глазами.
