# Сценарий 01: слепая диагностика

`inject.sh` включает одну из четырёх поломок (`slow`, `error500`,
`badjson`, `wrongct`) случайным образом. Ваша задача — определить какую,
пользуясь ТОЛЬКО клиентскими командами (curl/jq), как при реальном
инциденте.

Правила:
- `GET /api/v1/_lab/state` — нельзя (это «внутренности», которых у вас
  не будет);
- логи стенда — можно (у L2 обычно есть доступ к логам своей системы);
- диагноз — одно слово первой строкой в `/tmp/api-lab/m06-diagnosis.txt`;
- проверка и уборка — `bash check.sh`.

Если ошиблись — `inject.sh` можно запускать снова сколько угодно раз
(режим каждый раз случайный).

Методичка различения четырёх режимов: `../../solutions/01-blind-diagnosis/`
(сначала попробуйте без неё — на собеседовании подсказок не будет).
