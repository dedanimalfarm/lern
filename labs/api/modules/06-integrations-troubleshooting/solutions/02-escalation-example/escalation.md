# Эскалация: Helpdesk API отдаёт 500 на /api/v1/tickets

## Симптом
С 2026-06-11 01:14 UTC все обращения к заявкам через API завершаются
ошибкой 500. Затронуты: интеграция CRM, Postman-smoke смены, выгрузки
отчётов. Пользовательский веб-интерфейс не проверялся (нет доступа).

## Воспроизведение
```bash
curl -s -i http://localhost:8080/api/v1/tickets | head -1
# HTTP/1.1 500 Internal Server Error
curl -s http://localhost:8080/api/v1/tickets
# {"error":{"code":"internal_error","message":"unexpected error, see server logs"}}
```
Воспроизводится стабильно (10 из 10 попыток, 01:14–01:25 UTC).

## Ожидаемо / Факт
- Ожидаемо (по docs/openapi.yaml): 200 + `{"items":[...],"total":N,...}`.
- Факт: 500 + `internal_error` на ЛЮБОЙ запрос к /api/v1/tickets*
  (GET список, GET по id, POST).

## Что уже проверено
- `/health` — 200, время ответа <5 мс ⇒ процесс жив, деградация частичная;
- запрос корректен: тот же GET без изменений работал до 01:14 UTC
  (см. access-лог: `"GET /api/v1/tickets HTTP/1.1" 200` в 01:12,
  `... 500` начиная с 01:14);
- 4xx-причины исключены: код 5xx, тело прямо отсылает к логам сервера;
- access-лог сервера: все запросы к tickets с 01:14 — 500; stacktrace
  приложения в логе отсутствует (доступен только access-лог) — нужен
  доступ L3 к application-логу;
- со стороны клиентов ничего не менялось (коллекция и скрипты те же).

## Workaround
Нет. Создание заявок через API недоступно; приём заявок временно
переведён на почту (ручная регистрация после восстановления).
