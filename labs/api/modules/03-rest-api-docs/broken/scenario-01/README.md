# Сценарий 01: алерты мониторинга не превращаются в заявки

**Симптом:** мониторинг шлёт алерты, заявки в Helpdesk не появляются.
Скрипт интеграции — `sync-from-monitoring.sh`. Ошибок три, и все
находятся в документации (`docs/openapi.yaml`).

<details>
<summary>Подсказка 1 — что говорит первый же ответ</summary>

`HTTP/1.1 301 Moved Permanently` + `Location: /api/v1/tickets`.
Интеграция ходит в **v0**, который выведен из эксплуатации. Скрипт не
следует редиректам (и это típично для интеграций!) — поэтому «молча
ничего не создаётся». Поправьте версию пути. Почему не просто добавить
`-L`? POST при редиректе многие клиенты превращают в GET — надёжнее
ходить сразу по правильному адресу.
</details>

<details>
<summary>Подсказка 2 — после исправления пути</summary>

`422 validation_failed`. Читайте `details`: сервер перечисляет
неизвестные поля. Мониторинг говорит на своём словаре (`subject`,
`severity`), Helpdesk — на своём. Каком? Смотрите `components/schemas`
→ `TicketCreate` в доке. Сопоставьте: subject → ?, severity → ?
</details>

<details>
<summary>Подсказка 3 — последнее поле</summary>

`status` при создании задаёт СЕРВЕР (в доке: «Поле `status` при создании
задаётся сервером… и в теле запроса не допускается»). Просто уберите его.
</details>

## Критерий успеха

```bash
bash sync-from-monitoring.sh
# HTTP/1.1 201 Created
# {"id":NN,"title":"Алерт: диск заполнен на 95%","priority":"critical",...}
```
