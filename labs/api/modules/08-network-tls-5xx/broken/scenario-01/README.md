# Broken-сценарий: слепая диагностика 5xx за балансировщиком

Финальный экзамен модуля. Скрипт включает СЛУЧАЙНЫЙ серверный отказ из
четырёх — вы не знаете какой:

```bash
bash broken/scenario-01/inject.sh
# [OK] серверная поломка включена. Диагностируйте, НЕ заглядывая в _lab/state!
```

Диагностируйте **только клиентскими командами** (без `_lab/state` — в
реальном инциденте такой ручки у вас нет). Запишите диагноз — одно слово
`error502` / `error503` / `error504` / `flaky` — первой строкой в
`/tmp/api-lab/m08-5xx-diagnosis.txt`. Затем:

```bash
bash broken/scenario-01/check.sh
# [OK] диагноз верный: <mode> (поломка выключена)
```

Признаки, по которым различают (методичка — в
`solutions/01-blind-5xx/README.md`, сначала попробуйте сами):

- **Content-Type ответа** — наш JSON-конверт ошибки или `text/html`
  «балансировщика»? html = ответил не бэкенд.
- **Заголовок `Retry-After`** — кто его прислал, тот сам говорит, когда
  вернуться.
- **Поведение в СЕРИИ** — стабильный код на каждый запрос или «то 200,
  то 500»? Интермиттентность видна только на серии, не на одиночном curl.
