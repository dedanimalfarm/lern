# Сценарий 01: smoke-коллекция предыдущей смены вся красная

**Симптом:** импортируете `broken.postman_collection.json`, запускаете
Collection Runner — оба запроса падают.

Чините В POSTMAN (не правкой JSON-файла руками — хотя после модуля 02 вы
смогли бы и так): красный тест → смотрим фактический ответ → чиним запрос.

<details>
<summary>Подсказка 1 — «Список тикетов»</summary>

Тест ждёт 200, пришло **404**. Откройте вкладку Body ответа: сервер
называет путь, которого у него нет. Сравните URL запроса с рабочей
коллекцией `helpdesk.postman_collection.json` (или с докой): коллекция
называется во множественном числе.
</details>

<details>
<summary>Подсказка 2 — «Создать тикет»</summary>

Тест ждёт 201, пришло **415 Unsupported Media Type** (лестница из
модуля 01!). Посмотрите в запросе вкладку Body: тип стоит `raw → Text`,
поэтому Postman отправил `Content-Type: text/plain`. Переключите на
`raw → JSON` — Postman сам выставит правильный заголовок (проверьте это
во вкладке Headers: серый автозаголовок).
</details>

## Критерий успеха

Runner показывает все тесты зелёными. Экспортируйте починенную коллекцию:
правый клик по коллекции → **Export** → Collection v2.1 → сохранить как
`/tmp/api-lab/m04-fixed-collection.json` (из Windows-Postman сохраняйте в
`\\wsl$\...\tmp\api-lab\` или экспортируйте куда удобно и скопируйте в WSL).

Эталон: `../../solutions/01-fixed-collection/`.
