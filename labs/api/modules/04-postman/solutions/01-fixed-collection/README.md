# Решение: красная smoke-коллекция

Эталон — `fixed.postman_collection.json` (прогнан newman'ом:
3 assertions, 0 failed). Две починки:

1. **«Список тикетов»: 404 → 200.** URL вёл на `/api/v1/ticket`
   (единственное число). Коллекции ресурсов в REST именуются во
   множественном: `/api/v1/tickets`. Диагноз ставится по телу ответа —
   сервер прямо пишет `нет такого пути`.
2. **«Создать тикет»: 415 → 201.** Body был `raw → Text`, и Postman слал
   `Content-Type: text/plain`. Переключение на `raw → JSON` заставляет
   Postman автоматически выставить `application/json` (в JSON-файле
   коллекции это поле `options.raw.language`). Лестница ошибок из
   модуля 01: 415 = «формат не тот», смотреть Content-Type.

Мораль: «красный тест» в Postman диагностируется так же, как ошибка curl —
по статус-коду и телу фактического ответа, а не по тексту теста.
