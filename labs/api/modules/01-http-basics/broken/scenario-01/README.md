# Сценарий 01: скрипт создания заявки не работает

**Симптом:** `bash create-ticket.sh` не создаёт заявку. Пользователь
прислал вывод: `{"error":{"code":"not_found",...}}`.

В скрипте **три независимые ошибки**. Каждая проявляется СВОИМ
статус-кодом — чините по одной и перезапускайте.

## Ход диагностики (разворачивайте подсказки по необходимости)

<details>
<summary>Подсказка 1 — с чего начать</summary>

Добавьте к curl флаг `-i` (или `-w '%{http_code}\n'`), чтобы видеть
статус-код, а не только тело. Первый код будет **404**. Читайте `message`
в теле ошибки: сервер прямо говорит, какого пути у него нет. Сверьте путь
с рабочими командами из Части 2 README.
</details>

<details>
<summary>Подсказка 2 — после исправления пути</summary>

Теперь **415 Unsupported Media Type**. Вспомните «лестницу» ошибок POST из
Части 3: чего не хватает запросу, чтобы сервер вообще стал читать тело?
</details>

<details>
<summary>Подсказка 3 — после исправления заголовка</summary>

Теперь **400 invalid_json**, в `details.parser` — «Expecting property name
enclosed in double quotes». Посмотрите внимательно на кавычки внутри JSON.
JSON признаёт ТОЛЬКО двойные кавычки (подробно — модуль 02).
</details>

## Критерий успеха

```bash
bash create-ticket.sh
# HTTP/1.1 201 Created  (если оставили -i)
# {"id":NN,"title":"Заявка из скрипта",...,"status":"open","priority":"high",...}

curl -s -G http://localhost:8080/api/v1/tickets --data-urlencode 'q=Заявка из скрипта' | jq '.total'
# 1 (или больше, если запускали несколько раз — кстати, почему? см. модуль 06)
```
