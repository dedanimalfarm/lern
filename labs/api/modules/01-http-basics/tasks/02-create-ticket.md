# 02 — Создать тикет и проверить его (POST → 201 → Location → GET)

## Задача
1. POST'ом создать тикет, в `title` которого есть подстрока `m01`
   (например, «Проверка связи m01»).
2. Убедиться, что ответ — `201 Created`, и взять путь из заголовка `Location`.
3. GET'ом по этому пути получить тикет обратно.
4. Найти его поиском: `GET /api/v1/tickets?q=m01`.

## Проверка
```bash
curl -s 'http://localhost:8080/api/v1/tickets?q=m01' | jq '.total'   # >= 1
```

## Ожидаемый результат
Поиск `?q=m01` возвращает `total >= 1`, ваш тикет имеет `status: "open"`
(статус при создании задаёт сервер).
