# 02 — Лимиты ресурсов защищают хост

## Задача
Дать контейнеру лимит памяти и убедиться, что утечка памяти внутри получает OOM
**в контейнере** (cgroup из этапа 04), не затрагивая хост.

## Проверка
```bash
sudo ./11-capstone/mycontainer.sh run -m 32M alpine -- sh -c 'echo до; tail /dev/zero; echo после'
```

## Ожидаемый результат
```
до
Killed              # хог tail /dev/zero упёрся в memory.max=32M → OOM убил ЕГО
после               # shell пережил (OOM убивает самый большой процесс в cgroup)
```
Процесс в контейнере не смог выйти за 32M — cgroup-лимит сработал. Без лимита такая
утечка съела бы RAM хоста; с `-m` она ограничена контейнером. Так `--memory` у
Docker защищает хост. Аналогично `-p` (pids.max) останавливает fork-бомбу.
