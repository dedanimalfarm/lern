## Теоретическая часть

В Bash существуют различные механизмы для управления потоком выполнения: ветвления и циклы.

### Условный оператор `if`
Позволяет выполнять команды в зависимости от истинности условия.
```bash
if [ -f "file.txt" ]; then
    echo "Файл существует"
elif [ -d "dir" ]; then
    echo "Это директория"
else
    echo "Не найдено"
fi
```
Обратите внимание на пробелы после `[` и перед `]`. Они обязательны!

### Оператор `case`
Удобен для множественного выбора.
```bash
case "$1" in
    start)
        echo "Запуск..."
        ;;
    stop)
        echo "Остановка..."
        ;;
    *)
        echo "Неизвестная команда"
        ;;
esac
```

### Цикл `for`
Используется для итерации по списку значений или диапазону.
```bash
for i in 1 2 3 4 5; do
    echo "Число: $i"
done

# Использование диапазона
for i in {1..5}; do
    echo "Число: $i"
done
```

### Цикл `while`
Выполняется, пока условие истинно.
```bash
count=1
while [ $count -le 5 ]; do
    echo "Count: $count"
    ((count++))
done
```

### Цикл `until`
Выполняется, пока условие ложно.
```bash
count=1
until [ $count -gt 5 ]; do
    echo "Count: $count"
    ((count++))
done
```

## Troubleshooting — частые проблемы

1. **Ошибка `[: missing ]` или `command not found` в условии `if`**
   - **Причина:** Отсутствуют пробелы вокруг квадратных скобок `[ ]` или вокруг операторов сравнения (`=`, `==`, `-eq`).
   - **Решение:** Пишите `if [ "$var" = "value" ]; then`, а не `if ["$var"="value"]; then`.

2. **Скрипт зависает при использовании цикла `while` или `until`**
   - **Причина:** Забыли добавить шаг изменения переменной в теле цикла (например, `((count++))`), из-за чего цикл стал бесконечным.
   - **Решение:** Убедитесь, что условие выхода из цикла рано или поздно наступит.

3. **Скрипт не запускается (Permission denied)**
   - **Причина:** Файл не имеет прав на выполнение.
   - **Решение:** Выполните `chmod +x имя_скрипта.sh`.

4. **Ошибка `syntax error: unexpected end of file`**
   - **Причина:** Забыли закрыть цикл (`done`), условный оператор (`fi`) или `case` (`esac`).
   - **Решение:** Внимательно проверьте структуру скрипта. Полезно соблюдать отступы для каждого вложенного блока.
