# Junior DevOps Screening — учебный сборник лаб

Сборник собран из тестового задания на позицию **Junior DevOps под руководством lead-инженера**. Источник тестового — `/root/test-job` (4 части: Ansible / Observability / Secrets / Документация).

В отличие от самого тестового, этот сборник — **полноценный учебный курс**. Цель: пройдя его, кандидат получает **все базовые навыки по темам из вакансии**, а не только умеет «закрыть тестовое». В каждой лабе есть теоретический ввод, ручная отработка темы, автоматизация и acceptance criteria.

В конце — [`FINAL-TEST/`](./FINAL-TEST/), где темы склеиваются в исходное тестовое задание из вакансии (60–90 минут).

## Структура каждой лабы

Все лабы оформлены по одному шаблону:

1. **Цель и навыки** — что отрабатываем и что ты должен уметь после.
2. **Теоретический минимум** — короткая справка по теме (без воды, со ссылками на `man` и официальные доки).
3. **Базовая отработка** — ручные шаги, чтобы прочувствовать тему «на руках».
4. **Расширенная отработка** — автоматизация / отказоустойчивость / реалистичный сценарий.
5. **Acceptance criteria** — чек-лист, по которому ты сам себя проверишь.
6. **Что обсудить на ревью** — вопросы, которые гарантированно зададут на интервью.
7. **Грабли** — самые частые ошибки и их симптомы.

## Требования к стенду

- **ОС:** Ubuntu 22.04 или 24.04 LTS Server (курс проверен на 24.04 на EC2).
- **Ресурсы (комфортно):** 2 vCPU / 2 ГБ RAM / 10 ГБ диска.
- **Ресурсы (минимум):** 1 vCPU / 1 ГБ RAM — тоже работает, но Prometheus + Grafana + Vault + Loki одновременно держать нельзя; в каждой лабе есть блок «Как погасить» — пользуйся им между частями. На ≥2 ГБ RAM можно держать всё вместе.
- Для [04](./04-virtualization/) и [09](./09-zfs-snapshots/) хорошо иметь 4 ГБ RAM и доп. raw-диск 5–10 ГБ (на EC2 — EBS volume; иначе loop-файл).
- **Сеть:** исходящий доступ в интернет; входящий 22/tcp с твоего IP.
- **Контрольная машина:** `ansible-core ≥ 2.14`, `ssh`, `git`, `curl`, `jq`. Под Windows — WSL2.

> **Текущий референс-стенд** (если тебе его собрали заранее): AWS EC2 t3.micro, Ubuntu 24.04.4 LTS, пользователь `ubuntu`, ключ `~/.ssh/vast`, NOPASSWD-sudo. IP уточни у того, кто поднимал.

Полный чек-лист — в [`00-setup/README.md`](./00-setup/README.md).

## Как пользоваться

1. Подними стенд: [`00-setup`](./00-setup/).
2. Иди по лабам по порядку — каждая опирается на предыдущую.
3. После каждой лабы делай **атомарный коммит** с осмысленным сообщением.
4. В конце пройди [`FINAL-TEST/`](./FINAL-TEST/) — это то же самое тестовое из вакансии, но теперь ты должен уложиться в 60–90 минут.

## Критерии итоговой оценки

Полная рубрика — [`EVALUATION.md`](./EVALUATION.md). Кратко:

- **Идемпотентность** автоматизации.
- **Безопасность по умолчанию**: ssh без паролей, ufw on, секретов в репо нет.
- **Воспроизводимость**: `git clone && ансибл-прогон` поднимает всё с нуля.
- **Документация**: README отвечает на «как поднять», «как проверить», «что я бы сделал на prod».
- **Git-гигиена**: атомарные коммиты, осмысленные сообщения, PR с описанием.
