# Рубрика самопроверки

Этот документ — то, по чему я (или ревьюер) проверит твою работу. Запускай его как чек-лист после каждой лабы и обязательно — перед сдачей [FINAL-TEST](./FINAL-TEST/).

## Принципы оценки

1. **Реальность важнее презентации.** Лучше — «работает, но коряво в одном месте», чем «красиво, но при первом прогоне падает».
2. **Идемпотентность — не опция.** Автоматизация, которая ломает второй прогон, не считается.
3. **Без хардкода секретов.** Любое `password=...` в git = провал.
4. **Документация = код.** README/runbook/ADR обновляются в том же PR, что и поведение.
5. **Аккуратность с production.** Если в лабе есть «опасная» команда — должна быть подсказка про rollback (`netplan try`, `at`-rollback ufw, `sshd -t`).

## Ось 1. Идемпотентность (вес 20)

| Балл | Критерий |
|------|----------|
|  20  | Все плейбуки после первого прогона дают `changed=0`. `--check --diff` чистый. |
|  15  | 1 модуль фейлит идемпотентность, но видно почему и есть план фикса (комментарий/TODO). |
|  10  | Идемпотентность есть «глобально», но `lineinfile`/`shell` без `creates:` встречаются. |
|   5  | Плейбук работает, но `changed > 0` на каждый прогон без видимой причины. |
|   0  | Плейбук вообще не запускается дважды (state hardcoded). |

## Ось 2. Безопасность (вес 15)

| Балл | Критерий |
|------|----------|
|  15  | Пароли/токены в Vault или ansible-vault; ufw enabled; ssh без password; root-login no; нет «висящих» NOPASSWD на ALL без обоснования. |
|  10  | Один из пунктов нарушен с явным TODO. |
|   5  | Пароли в env-vars docker-compose без шифрования, но **не в git**. |
|   0  | `password: changeme` или приватный ключ в git-репе. |

## Ось 3. Воспроизводимость (вес 15)

| Балл | Критерий |
|------|----------|
|  15  | `git clone && make up` (или эквивалентная одна команда) поднимает всё. Документация это утверждает, и оно правда. |
|  10  | Требуется руками выполнить ≤3 шага, но они описаны. |
|   5  | Поднятие требует чтения скрытых сценариев, проб и ошибок. |
|   0  | «У меня работает». |

## Ось 4. Документация (вес 10)

| Балл | Критерий |
|------|----------|
|  10  | README с разделами: как поднять, как проверить, как удалить, что на проде. ASCII-схема. ADR на одно ключевое решение. |
|   7  | Все вопросы выше отвечены, но без ADR/схемы. |
|   4  | README есть, но «как удалить» / «на проде» отсутствуют. |
|   0  | README — заголовок и три строки. |

## Ось 5. Git-гигиена (вес 10)

| Балл | Критерий |
|------|----------|
|  10  | Атомарные коммиты, Conventional Commits, нет «wip» и «.» в логе, нет утечек (`.env`, ключи). |
|   7  | Атомарность есть, сообщения иногда вольные. |
|   4  | Большие коммиты «всё подряд», но без утечек. |
|   0  | Утечка секретов в репе, либо force-push в main. |

## Ось 6. Архитектурное мышление (вес 10)

«Что я бы сделал на проде» в README, ADR, обсуждение trade-offs.

| Балл | Критерий |
|------|----------|
|  10  | Чётко формулирует ограничения учебного стенда vs prod (HA, secret rotation, retention, capacity planning). |
|   7  | Понимает 2–3 ключевых ограничения. |
|   4  | Поверхностное «надо мониторинг и бэкап». |
|   0  | Нет вообще. |

## Ось 7. Чистота (вес 10)

| Балл | Критерий |
|------|----------|
|  10  | `docker compose down`, `losetup -d`, `zpool export` — есть в каждой лабе и работают. После прогона всех лаб VM остаётся в чистом состоянии. |
|   7  | Чистка местами автоматизирована, местами в README текстом. |
|   3  | Тестовый мусор копится. |
|   0  | После лаб VM не загружается, либо `df` показывает 100%. |

## Ось 8. Сетевая безопасность (вес 10)

| Балл | Критерий |
|------|----------|
|  10  | `ufw` сужен; всё, что наружу нужно ставить (Grafana UI), ходит **через ssh-туннель**, а не публично. AWS SG минимальная. |
|   5  | Часть портов открыта наружу публично, без auth (Grafana без admin password). |
|   0  | `0.0.0.0/0` на 8200/9090/3000 одновременно. |

## Итоговый счёт

Сложи баллы по 8 осям → максимум 100. Пороги:

- **≥85** — отличный джун, готовый сразу к проду под присмотром.
- **70–84** — крепкий джун; есть пара слабых мест, но они конкретны.
- **55–69** — есть фундамент, нужно дотянуть **3 конкретные оси** перед собеседованием.
- **<55** — пройди соответствующие лабы ещё раз, обращай внимание на разделы «Что обсудить на ревью».

## Само-проверка перед сдачей

```bash
# 1. Проверка идемпотентности
ansible-playbook playbooks/site.yml | tee /tmp/run1.txt
ansible-playbook playbooks/site.yml | tee /tmp/run2.txt
grep "changed=0" /tmp/run2.txt && echo "IDEMPOTENT" || echo "NOT IDEMPOTENT"

# 2. Поиск утечек секретов
git ls-files | xargs grep -nE -i 'password|secret|api[_-]?key|token' \
  | grep -v -E '(README|docs/|\.md:|defaults/|TEMPLATE)' \
  | head -20
# Каждое попадание — глазами проверь, что это не реальный секрет.

# 3. Проверка чистки
docker ps -a; docker volume ls; docker network ls
zpool list 2>/dev/null; losetup -l 2>/dev/null
df -h /
# Ничего лишнего после teardown.

# 4. README clone-and-run
# Открой README в режиме «незнакомца» и пройди по шагам без отступлений.
```
