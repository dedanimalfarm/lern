# Задание 1: Резервное копирование и восстановление таблиц

База данных: `shop_db`

## Шаги выполнения

1. **Создание текстового бэкапа отдельной таблицы:**
   Сделайте резервную копию только таблицы `users` базы данных `shop_db` в текстовом (Plain SQL) формате. Сохраните её в файл `/tmp/users_backup.sql`.
   *(Подсказка: используйте флаг `-t users` для указания таблицы и `-F p` для выбора формата).*

2. **Создание полного сжатого бэкапа базы данных:**
   Сделайте полную резервную копию базы данных `shop_db` в сжатом бинарном (Custom) формате. Сохраните её в файл `/tmp/shop_db.dump`.
   *(Подсказка: используйте формат `-F c`).*

3. **Симуляция аварии (Удаление данных):**
   Удалите таблицу `order_items` из базы данных `shop_db` с помощью psql (или через терминал БД):
   ```sql
   DROP TABLE order_items;
   ```
   Убедитесь, что таблица исчезла и запросы к ней выдают ошибку.

4. **Выборочное восстановление таблицы:**
   Восстановите из созданного бинарного бэкапа `/tmp/shop_db.dump` **только** удаленную таблицу `order_items`, не затрагивая остальные данные.
   *(Подсказка: используйте утилиту `pg_restore` с флагом `-t order_items`).*
   Убедитесь, что таблица `order_items` снова появилась в БД и содержит все данные.

## 🏆 Задания для самостоятельной работы

Попробуйте написать команды для следующих задач (решения не предоставляются, проверьте себя в терминале):

1. **Исследование структуры бэкапа:** 
   * Откройте файл `/tmp/users_backup.sql` с помощью текстового редактора или команды `cat`. Найдите в нем SQL-команду создания таблицы `CREATE TABLE users` и команды наполнения данных.
   * Попробуйте аналогично открыть `/tmp/shop_db.dump`. Почему его содержимое не читается?
2. **Просмотр оглавления архива:** Выведите список всех объектов (таблиц, индексов, триггеров), хранящихся внутри сжатого бэкапа `/tmp/shop_db.dump`, не восстанавливая его.
   *(Подсказка: используйте `pg_restore --list` или `-l`).*
3. **Автоматизация бэкапов (Bash/Cron):** Напишите однострочный bash-скрипт (или cron-запись), который делает бэкап базы данных `shop_db` каждый день в 00:00, автоматически подставляя текущую дату в имя файла (например, `/tmp/shop_db_2026-06-13.dump`).
