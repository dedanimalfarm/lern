# Задание 1: Резервное копирование и восстановление таблиц

База данных: `pagila`

## Шаги выполнения

1. **Создание текстового бэкапа отдельной таблицы:**
   Сделайте резервную копию только таблицы `actor` базы данных `pagila` в текстовом (Plain SQL) формате. Сохраните её в файл `/tmp/actor_backup.sql`.
   *(Подсказка: используйте флаг `-t actor` для указания таблицы и `-F p` для выбора формата).*

2. **Создание полного сжатого бэкапа базы данных:**
   Сделайте полную резервную копию базы данных `pagila` в сжатом бинарном (Custom) формате. Сохраните её в файл `/tmp/pagila.dump`.
   *(Подсказка: используйте формат `-F c`).*

3. **Симуляция аварии (Удаление данных):**
   Удалите таблицу `film_category` из базы данных `pagila` с помощью psql (или через терминал БД):
   ```sql
   DROP TABLE film_category;
   ```
   Убедитесь, что таблица исчезла и запросы к ней выдают ошибку.

4. **Выборочное восстановление таблицы:**
   Восстановите из созданного бинарного бэкапа `/tmp/pagila.dump` **только** удаленную таблицу `film_category`, не затрагивая остальные данные.
   *(Подсказка: используйте утилиту `pg_restore` с флагом `-t film_category`).*
   Убедитесь, что таблица `film_category` снова появилась в БД и содержит все данные.

## 🏆 Задания для самостоятельной работы

Попробуйте написать команды для следующих задач (решения не предоставляются, проверьте себя в терминале):

1. **Исследование структуры бэкапа:** 
   * Откройте файл `/tmp/actor_backup.sql` с помощью текстового редактора или команды `cat`. Найдите в нем SQL-команду создания таблицы `CREATE TABLE actor` и команды наполнения данных.
   * Попробуйте аналогично открыть `/tmp/pagila.dump`. Почему его содержимое не читается?
2. **Просмотр оглавления архива:** Выведите список всех объектов (таблиц, индексов, триггеров), хранящихся внутри сжатого бэкапа `/tmp/pagila.dump`, не восстанавливая его.
   *(Подсказка: используйте `pg_restore --list` или `-l`).*
3. **Автоматизация бэкапов (Bash/Cron):** Напишите однострочный bash-скрипт (или cron-запись), который делает бэкап базы данных `pagila` каждый день в 00:00, автоматически подставляя текущую дату в имя файла (например, `/tmp/pagila_2026-06-13.dump`).
