# Модуль 4: Управление данными (DML)

## 📖 Теоретическая часть

До сих пор мы только читали данные. Язык управления данными (Data Manipulation Language, DML) позволяет изменять состояние базы данных.

### 1. Добавление данных (INSERT)
Используется для добавления одной или нескольких новых строк в таблицу.
```sql
INSERT INTO users (name, email, registration_date) 
VALUES ('Джон Доу', 'john@example.com', '2023-10-01');

-- Множественная вставка
INSERT INTO categories (name) 
VALUES ('Одежда'), ('Обувь');
```
Если вы не указываете значение для поля (например, `id`), СУБД сгенерирует его автоматически (если поле `SERIAL` или имеет `DEFAULT`).

### 2. Обновление данных (UPDATE)
Обновляет существующие записи.
```sql
UPDATE products 
SET price = 150000, stock_quantity = 5 
WHERE name = 'Ноутбук Dell XPS';
```
> ⚠️ **Критически важно:** Если вы забудете секцию `WHERE` в запросе `UPDATE`, база данных обновит **ВСЕ** строки в таблице!

### 3. Удаление данных (DELETE)
Удаляет строки из таблицы.
```sql
DELETE FROM orders WHERE status = 'CANCELLED';
```
Как и в случае с `UPDATE`, отсутствие `WHERE` приведет к полному удалению всех данных в таблице.

### 4. Ограничения целостности и каскадное удаление
В реляционных базах таблицы связаны. Если вы попытаетесь удалить пользователя, у которого есть заказы, PostgreSQL выдаст ошибку `foreign key constraint violation`, чтобы предотвратить появление "осиротевших" заказов.
Чтобы удалять записи вместе со связанными дочерними строками, при создании таблиц используют ограничение `ON DELETE CASCADE`.
