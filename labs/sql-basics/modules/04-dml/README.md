# Модуль 4: Управление данными (DML)

## 📖 Теоретическая часть (Что нужно знать)

Перед выполнением практических заданий необходимо самостоятельно изучить следующие темы:
- Добавление новых строк в таблицу: `INSERT INTO ... VALUES ...`.
- Множественная вставка данных (Bulk Insert).
- Обновление существующих записей: `UPDATE ... SET ... WHERE ...`.
- Удаление строк: `DELETE FROM ... WHERE ...`.
- Ограничения целостности и поведение при удалении связанных данных (`ON DELETE CASCADE`, `RESTRICT`).
- Использование оператора `RETURNING` в PostgreSQL для возврата измененных строк.

---

## ❓ Вопросы для самопроверки

1. Чем `DELETE FROM table;` отличается от `TRUNCATE TABLE table;` с точки зрения производительности и логики БД?
2. Что такое `RETURNING`, и как с его помощью получить сгенерированный базой `id` только что вставленной строки?
3. Представьте, что вы случайно запустили `UPDATE` без `WHERE` внутри транзакции (`BEGIN;`). Как можно спасти данные, пока транзакция не закрыта?
4. Что произойдет при попытке вставить новую строку, если значение в столбце с ограничением `UNIQUE` уже существует?
