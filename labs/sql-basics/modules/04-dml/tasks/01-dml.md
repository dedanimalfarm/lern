# Задание 1: Изменение данных

База данных: `shop_db`

## Шаги выполнения

1. Добавьте нового пользователя с вашим именем и email в таблицу `users`.
2. Магазин устроил распродажу! Обновите цену всех товаров в категории 'Аксессуары' (уменьшите цену на 10%). Подсказка: `SET price = price * 0.9`.
3. Заказ №5 был отменен (`status = 'CANCELLED'`). Удалите его из таблицы `orders` (он больше не нужен в базе).
   *Примечание: таблица `order_items` имеет настройку `ON DELETE CASCADE`, поэтому детали заказа удалятся автоматически вместе с самим заказом.*

## 🏆 Задания для самостоятельной работы

Попробуйте написать SQL-запросы для следующих задач (решения не предоставляются, проверьте себя в консоли psql):

1. **Массовое обновление остатков:** Поступила новая партия товаров. Увеличьте количество на складе (`stock_quantity`) на `20` для всех товаров категории `'Электроника'`.
2. **Эксперимент с транзакцией (Rollback):** Откройте транзакцию (`BEGIN;`), добавьте в нее тестового пользователя с почтой `'ghost@example.com'`, затем сделайте откат транзакции (`ROLLBACK;`). Выполните `SELECT` и убедитесь, что пользователь не сохранился в БД.
3. **Проверка целостности данных:** Попробуйте удалить пользователя с `id = 1` напрямую (`DELETE FROM users WHERE id = 1;`). 
   * Какую ошибку возвращает СУБД? 
   * Почему это происходит? (Подсказка: таблица `orders` ссылается на `users(id)`, но в её внешнем ключе не настроено поведение `ON DELETE CASCADE`).
