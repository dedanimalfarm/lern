# Задание 1: Изменение данных

База данных: `pagila`

## Шаги выполнения

1. Добавьте нового актера с вашим именем и фамилией в таблицу `actor` (поля `first_name`, `last_name`).
2. Магазин устроил распродажу! Обновите стоимость аренды (`rental_rate`) всех фильмов с рейтингом 'G' (уменьшите цену на 10%). Подсказка: `SET rental_rate = rental_rate * 0.9`.
3. Платеж №10000 был проведен ошибочно. Напишите запрос для его удаления из таблицы `payment` (`payment_id = 10000`).

## 🏆 Задания для самостоятельной работы

Попробуйте написать SQL-запросы для следующих задач (решения не предоставляются, проверьте себя в консоли psql):

1. **Массовое обновление:** Поступила новая директива. Увеличьте срок аренды (`rental_duration`) на `1` день для всех фильмов с рейтингом `'R'`.
2. **Эксперимент с транзакцией (Rollback):** Откройте транзакцию (`BEGIN;`), добавьте в нее тестового актера с именем `'GHOST'`, затем сделайте откат транзакции (`ROLLBACK;`). Выполните `SELECT` и убедитесь, что актер не сохранился в БД.
3. **Проверка целостности данных:** Попробуйте удалить клиента с `customer_id = 1` напрямую (`DELETE FROM customer WHERE customer_id = 1;`). 
   * Какую ошибку возвращает СУБД? 
   * Почему это происходит? (Подсказка: таблица `rental` ссылается на `customer(customer_id)`, но в её внешнем ключе не настроено поведение `ON DELETE CASCADE`).
