# Задание 1: Статистика магазина

База данных: `pagila`

## Шаги выполнения

1. Подсчитайте общее количество аренд в системе (таблица `rental`).
2. Найдите среднюю арендную ставку (`rental_rate`) фильмов с каждым рейтингом (таблица `film`). Используйте `GROUP BY`.
3. Подсчитайте, сколько аренд сделал каждый клиент (выведите `customer_id`, имя `first_name`, фамилию `last_name` и количество аренд). Сгруппируйте по идентификатору и ФИО.
4. Выведите список клиентов (`customer_id`, имя, фамилия), которые сделали **более 30** аренд (используйте `HAVING`).

## 🏆 Задания для самостоятельной работы

Попробуйте написать SQL-запросы для следующих задач (решения не предоставляются, проверьте себя в консоли psql):

1. **Группировка с фильтрацией строк:** Посчитайте количество фильмов каждого рейтинга, у которых стоимость аренды (`rental_rate`) превышает `1.00` (условие на цену должно отрабатывать до группировки).
2. **Суммарный аудит запасов:** Выведите рейтинги фильмов, суммарная стоимость возмещения которых (вычисляется как сумма `replacement_cost`) превышает `4 000`. Выведите рейтинг и итоговую сумму.
3. **Активность по дням:** Найдите день (дату), в который было совершено наибольшее количество аренд в магазине. Используйте группировку через `date_trunc('day', rental_date)`, сортировку по количеству аренд и `LIMIT 1`.
