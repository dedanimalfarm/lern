# Задание 1: Статистика магазина

База данных: `shop_db`

## Шаги выполнения

1. Подсчитайте общее количество заказов в системе (таблица `orders`).
2. Найдите среднюю цену товаров в каждой категории (таблица `products`). Используйте `GROUP BY`.
3. Подсчитайте, сколько заказов сделал каждый пользователь (нужно соединить `users` и `orders` и сгруппировать по имени).
4. Выведите имена только тех пользователей, которые сделали **более одного** заказа (используйте `HAVING`).

## 🏆 Задания для самостоятельной работы

Попробуйте написать SQL-запросы для следующих задач (решения не предоставляются, проверьте себя в консоли psql):

1. **Группировка с фильтрацией строк:** Посчитайте количество товаров в каждой категории, у которых цена превышает `10000` (условие на цену должно отрабатывать до группировки).
2. **Суммарный аудит запасов:** Выведите категории товаров, суммарная стоимость запасов которых (вычисляется как `цена * stock_quantity` по каждому товару) превышает `1 000 000`. Выведите категорию и итоговую сумму запасов.
3. **Активность по дням:** Найдите день (дату), в который было совершено наибольшее количество заказов в магазине. Используйте группировку через `date_trunc('day', order_date)`, сортировку по количеству заказов и `LIMIT 1`.
