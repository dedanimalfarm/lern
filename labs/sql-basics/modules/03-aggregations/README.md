## Часть 1: Агрегатные функции

Агрегатные функции вычисляют единственное результирующее значение для множества строк.
Базовые функции: `COUNT`, `SUM`, `AVG` (среднее), `MIN`, `MAX`.

```sql
-- Подсчет общей суммы всех платежей и среднего платежа
SELECT SUM(amount), AVG(amount) 
FROM payment;
```

## Часть 2: Поведение NULL в агрегациях

При работе с агрегациями нужно помнить критически важное правило: **все агрегатные функции (кроме `COUNT(*)`) игнорируют `NULL`-значения**.

1. **Разница `COUNT(*)` и `COUNT(колонка)`:**
   * `COUNT(*)` считает общее количество строк в таблице, даже если они содержат `NULL`.
   * `COUNT(email)` посчитает количество строк, где колонка `email` **заполнена** (не `NULL`).
2. **Математические функции (`SUM`, `AVG`, `MIN`, `MAX`):**
   * Если в колонке с ценами есть `NULL`, функция `AVG(amount)` рассчитает среднее только по строкам с заполненной ценой. Строки с `NULL` не будут участвовать ни в сумме, ни в делителе среднего значения.
   * Если таблица пуста или все строки в группе содержат `NULL`, то `SUM` и `AVG` вернут `NULL` (а не `0`).

## Часть 3: Разделение на группы

Конструкция `GROUP BY` позволяет разбить таблицу на сегменты (группы) по значениям определенной колонки, и применить агрегатную функцию к каждому сегменту отдельно.

```sql
-- Средняя стоимость аренды фильмов в разрезе рейтингов
SELECT rating, AVG(rental_rate) AS average_rate 
FROM film 
GROUP BY rating;
```

## Часть 4: Группировка по времени (date_trunc) для DevOps

Для DevOps-инженеров и инженеров поддержки одна из самых частых задач — сбор метрик по времени (например, "сколько ошибок произошло в час" или "сколько аренд оформляется в сутки" для Grafana-дашбордов).

В PostgreSQL для округления временных меток до нужного интервала (минута, час, день, месяц) используется функция `date_trunc(интервал, колонка)`:

```sql
-- Округление до часа (отбросит минуты и секунды):
-- '2005-05-24 22:54:33' -> '2005-05-24 22:00:00'
SELECT date_trunc('hour', rental_date) AS rental_hour, COUNT(*) AS count
FROM rental
GROUP BY rental_hour
ORDER BY rental_hour;
```
Этот запрос группирует аренды по часовым интервалам, позволяя строить графики нагрузки на систему.

## Часть 5: Фильтрация после группировки

Секция `WHERE` не может использоваться для фильтрации результатов работы агрегатных функций, так как `WHERE` отрабатывает ДО того, как происходит агрегация. Для фильтрации финальных групп используется конструкция `HAVING`.

```sql
-- Найти рейтинги фильмов, где средняя стоимость аренды превышает 3.00
SELECT rating, AVG(rental_rate) 
FROM film 
GROUP BY rating 
HAVING AVG(rental_rate) > 3.00;
```

## Часть 6: Troubleshooting

**Сценарий:** Вы хотите узнать среднюю стоимость аренды фильмов по рейтингам и одновременно вывести название самого дорогого фильма. Вы пишете:
```sql
SELECT rating, title, AVG(rental_rate) FROM film GROUP BY rating;
```
СУБД возвращает ошибку `column "film.title" must appear in the GROUP BY clause or be used in an aggregate function`.

**Причина и решение:** Это фундаментальное правило SQL. Любая колонка в блоке `SELECT`, которая не обернута в агрегатную функцию (в данном случае `title`), ОБЯЗАНА присутствовать в блоке `GROUP BY`. СУБД физически не знает, какое именно название фильма из десятков с одинаковым рейтингом нужно вывести в итоговую строку группы. Для решения такой задачи требуются оконные функции (Window Functions), которые изучаются в продвинутых курсах.
