# Модуль 3: Агрегация и группировка данных

## 📖 Теоретическая часть

Часто нас интересуют не сами данные по отдельности, а сводная информация (статистика): суммы, средние значения, количество записей.

### 1. Агрегатные функции
Агрегатные функции принимают множество значений из столбца, а возвращают одно значение.
- `COUNT(*)` или `COUNT(col)` — количество строк.
- `SUM(col)` — сумма значений.
- `AVG(col)` — среднее значение.
- `MIN(col) / MAX(col)` — минимум и максимум.

```sql
-- Сколько всего товаров на складе?
SELECT SUM(stock_quantity) FROM products;

-- Какова средняя цена товара?
SELECT AVG(price) FROM products;
```

### 2. Группировка (GROUP BY)
Если мы хотим получить агрегацию не для всей таблицы, а "в разрезе" какого-то признака (например, среднюю цену не вообще всех товаров, а *для каждой категории*), мы используем `GROUP BY`.
```sql
SELECT category, AVG(price) 
FROM products 
GROUP BY category;
```
> **Важное правило SQL:** Любой столбец в `SELECT`, который не обернут в агрегатную функцию, **обязан** быть перечислен в секции `GROUP BY`.

### 3. Фильтрация групп (HAVING)
Как отфильтровать результаты агрегации? Например, показать только те категории, где средняя цена больше 1000. 
Секция `WHERE` здесь не сработает, так как она фильтрует строки **до** группировки. Для фильтрации **после** группировки используется `HAVING`:
```sql
SELECT category, AVG(price) 
FROM products 
GROUP BY category
HAVING AVG(price) > 1000;
```
