# Модуль 3: Агрегация и группировка данных

## 📖 Теоретическая часть (Что нужно знать)

Перед выполнением практических заданий необходимо самостоятельно изучить следующие темы:
- Работа с агрегатными функциями: `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`.
- Группировка строк с помощью `GROUP BY`.
- Золотое правило группировки: почему все выбранные (неагрегированные) поля должны присутствовать в `GROUP BY`.
- Разница между фильтрацией `WHERE` и фильтрацией `HAVING`.
- Поведение агрегатных функций со значениями `NULL`.

---

## ❓ Вопросы для самопроверки

1. В чем разница между `COUNT(*)` и `COUNT(column_name)`? Будут ли учитываться `NULL` значения?
2. Можно ли использовать `WHERE` и `HAVING` в одном запросе? Если да, в каком порядке они выполняются базой данных?
3. Что будет, если в `SELECT` указать столбец, не обернутый в агрегатную функцию, но не добавить его в `GROUP BY`?
4. Как агрегатная функция `SUM` реагирует, если в складываемом столбце попадается значение `NULL`?
