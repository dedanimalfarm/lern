## Часть 1: Агрегатные функции

Агрегатные функции вычисляют единственное результирующее значение для множества строк.
Базовые функции: `COUNT`, `SUM`, `AVG` (среднее), `MIN`, `MAX`.

```sql
-- Подсчет общей суммы всех заказов и средней стоимости позиции
SELECT SUM(price_per_unit * quantity), AVG(price_per_unit) 
FROM order_items;
```

## Часть 2: Поведение NULL в агрегациях

При работе с агрегациями нужно помнить критически важное правило: **все агрегатные функции (кроме `COUNT(*)`) игнорируют `NULL`-значения**.

1. **Разница `COUNT(*)` и `COUNT(колонка)`:**
   * `COUNT(*)` считает общее количество строк в таблице, даже если они содержат `NULL`.
   * `COUNT(email)` посчитает количество строк, где колонка `email` **заполнена** (не `NULL`).
2. **Математические функции (`SUM`, `AVG`, `MIN`, `MAX`):**
   * Если в колонке с ценами есть `NULL`, функция `AVG(price)` рассчитает среднее только по строкам с заполненной ценой. Строки с `NULL` не будут участвовать ни в сумме, ни в делителе среднего значения.
   * Если таблица пуста или все строки в группе содержат `NULL`, то `SUM` и `AVG` вернут `NULL` (а не `0`).

## Часть 3: Разделение на группы

Конструкция `GROUP BY` позволяет разбить таблицу на сегменты (группы) по значениям определенной колонки, и применить агрегатную функцию к каждому сегменту отдельно.

```sql
-- Средняя цена товаров в разрезе категорий
SELECT category, AVG(price) AS average_price 
FROM products 
GROUP BY category;
```

## Часть 4: Группировка по времени (date_trunc) для DevOps

Для DevOps-инженеров и инженеров поддержки одна из самых частых задач — сбор метрик по времени (например, "сколько ошибок произошло в час" или "сколько заказов создается в сутки" для Grafana-дашбордов).

В PostgreSQL для округления временных меток до нужного интервала (минута, час, день, месяц) используется функция `date_trunc(интервал, колонка)`:

```sql
-- Округление до часа (отбросит минуты и секунды):
-- '2026-06-13 14:35:12' -> '2026-06-13 14:00:00'
SELECT date_trunc('hour', order_date) AS order_hour, COUNT(*) AS count
FROM orders
GROUP BY order_hour
ORDER BY order_hour;
```
Этот запрос группирует заказы по часовым интервалам, позволяя строить графики нагрузки на систему.

## Часть 5: Фильтрация после группировки

Секция `WHERE` не может использоваться для фильтрации результатов работы агрегатных функций, так как `WHERE` отрабатывает ДО того, как происходит агрегация. Для фильтрации финальных групп используется конструкция `HAVING`.

```sql
-- Найти категории, где средняя цена товара превышает 10 000
SELECT category, AVG(price) 
FROM products 
GROUP BY category 
HAVING AVG(price) > 10000;
```

## Часть 6: Troubleshooting

**Сценарий:** Вы хотите узнать среднюю цену товаров в категории и одновременно вывести название самого дорогого товара. Вы пишете:
```sql
SELECT category, name, AVG(price) FROM products GROUP BY category;
```
СУБД возвращает ошибку `column "products.name" must appear in the GROUP BY clause or be used in an aggregate function`.

**Причина и решение:** Это фундаментальное правило SQL. Любая колонка в блоке `SELECT`, которая не обернута в агрегатную функцию (в данном случае `name`), ОБЯЗАНА присутствовать в блоке `GROUP BY`. СУБД физически не знает, какое именно имя товара из десятков в категории нужно вывести в итоговую строку группы. Для решения такой задачи требуются оконные функции (Window Functions), которые изучаются в продвинутых курсах.
