# Модуль 1: Основы выборки данных (SELECT)

## 📖 Теоретическая часть

Основой взаимодействия с любой реляционной базой данных является язык запросов SQL, а его главной командой — `SELECT`. Она позволяет читать данные из таблиц.

### 1. Базовый синтаксис
Запрос состоит из указания того, **что** мы хотим получить (какие столбцы) и **откуда** (из какой таблицы):
```sql
-- Выбрать все столбцы (звездочка означает "все")
SELECT * FROM products;

-- Выбрать только конкретные столбцы
SELECT name, price FROM products;
```

### 2. Фильтрация данных (WHERE)
Чтобы получить не все строки, а только удовлетворяющие условию, используется секция `WHERE`:
```sql
SELECT * FROM products WHERE price > 50000;
SELECT * FROM users WHERE name = 'Иван Иванов';
```
Операторы, доступные в `WHERE`:
- Равенство/неравенство: `=`, `<>`, `!=`, `>`, `<`, `>=`, `<=`
- Логические операторы: `AND`, `OR`, `NOT`
- Проверка на вхождение в список: `IN (val1, val2)`
- Диапазон: `BETWEEN min AND max`
- Поиск по шаблону (строки): `LIKE '%Иван%'` (где `%` заменяет любое количество символов).

### 3. Сортировка (ORDER BY) и Ограничение (LIMIT)
По умолчанию база данных не гарантирует порядок вывода строк. Если порядок важен, мы задаем его явно:
```sql
-- По возрастанию (ASC - по умолчанию)
SELECT name, price FROM products ORDER BY price ASC;

-- По убыванию (DESC)
SELECT name, price FROM products ORDER BY price DESC;
```
Для выдачи только первых `N` записей используют `LIMIT` (часто применяется вместе с `ORDER BY` для поиска "топ-N" записей):
```sql
-- Получить топ-5 самых дорогих товаров
SELECT * FROM products ORDER BY price DESC LIMIT 5;
```
