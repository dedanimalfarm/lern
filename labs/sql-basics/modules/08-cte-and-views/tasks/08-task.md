# Задание 8: CTE и Материализованные представления

База данных: `pagila`

## Шаги выполнения

Бизнес попросил создать "Витрину данных" (Data Mart), которая будет быстро возвращать статистику по покупателям: сколько платежей сделал каждый клиент, и какова дата его последнего платежа. Запрос тяжелый, поэтому витрину нужно сделать материализованной.

Напишите SQL-скрипт (в файл `solution.sql`), который выполняет следующие действия:

1. Используя конструкцию `CREATE MATERIALIZED VIEW`, создайте материализованное представление с именем `customer_stats`.
2. Представление должно содержать три колонки:
   * `customer_id` (идентификатор пользователя)
   * `total_payments` (общее количество платежей этого пользователя)
   * `last_payment_date` (дата последнего платежа `payment_date` для этого пользователя)
3. Выборку нужно строить на основе таблицы `payment`. Сгруппируйте данные по `customer_id`.
4. (Опционально) Создайте уникальный индекс на колонку `customer_id` внутри `customer_stats` (это нужно для `REFRESH CONCURRENTLY` в будущем). Индекс назовите `idx_customer_stats_customer_id`.

*Подсказка: не забудьте про `DROP MATERIALIZED VIEW IF EXISTS customer_stats;` в начале скрипта для идемпотентности.*

## 🏆 Задания для самостоятельной работы

1. **Рефреш кэша:** Вставьте новый платеж в таблицу `payment`. Сделайте `SELECT` из `customer_stats` и убедитесь, что новый платеж там не появился. Выполните `REFRESH MATERIALIZED VIEW customer_stats;` и проверьте снова!
2. **CTE:** Напишите запрос без создания `VIEW`, используя `WITH` (CTE). В первом шаге CTE посчитайте сумму платежей для каждого пользователя. В основном `SELECT` отфильтруйте только тех пользователей, у которых сумма платежей больше 100.
3. **Обычный VIEW:** Создайте обычный `VIEW` с именем `premium_films`, который возвращает только те фильмы из таблицы `film`, `rental_rate` которых больше 3. Убедитесь, что данные в нем всегда актуальны при изменении таблицы `film`.
