# Задание 8: CTE и Материализованные представления

База данных: `shop_db`

## Шаги выполнения

Бизнес попросил создать "Витрину данных" (Data Mart), которая будет быстро возвращать статистику по покупателям: сколько заказов сделал каждый клиент, и какова средняя сумма его чека. Запрос тяжелый, поэтому витрину нужно сделать материализованной.

Напишите SQL-скрипт (в файл `solution.sql`), который выполняет следующие действия:

1. Используя конструкцию `CREATE MATERIALIZED VIEW`, создайте материализованное представление с именем `customer_stats`.
2. Представление должно содержать три колонки:
   * `user_id` (идентификатор пользователя)
   * `total_orders` (общее количество заказов этого пользователя)
   * `last_order_date` (дата последнего заказа `order_date` для этого пользователя)
3. Выборку нужно строить на основе таблицы `orders`. Сгруппируйте данные по `user_id`.
4. (Опционально) Создайте уникальный индекс на колонку `user_id` внутри `customer_stats` (это нужно для `REFRESH CONCURRENTLY` в будущем). Индекс назовите `idx_customer_stats_user_id`.

*Подсказка: не забудьте про `DROP MATERIALIZED VIEW IF EXISTS customer_stats;` в начале скрипта для идемпотентности.*

---

## 🏆 Задания для самостоятельной работы

1. **Рефреш кэша:** Вставьте новый заказ в таблицу `orders`. Сделайте `SELECT` из `customer_stats` и убедитесь, что новый заказ там не появился. Выполните `REFRESH MATERIALIZED VIEW customer_stats;` и проверьте снова!
2. **CTE:** Напишите запрос без создания `VIEW`, используя `WITH` (CTE). В первом шаге CTE посчитайте сумму заказов для каждого пользователя. В основном `SELECT` отфильтруйте только тех пользователей, у которых сумма заказов больше 20000.
3. **Обычный VIEW:** Создайте обычный `VIEW` с именем `active_products`, который возвращает только те товары из таблицы `products`, цена которых больше 1000. Убедитесь, что данные в нем всегда актуальны при изменении таблицы `products`.
