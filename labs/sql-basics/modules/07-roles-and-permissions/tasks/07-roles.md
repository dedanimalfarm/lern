# Задание 7: Управление доступом (Roles & Permissions)

База данных: `shop_db`

## Шаги выполнения

Вам необходимо создать специальную роль и пользователя для системы аналитики, соблюдая принцип наименьших привилегий (Least Privilege).

Напишите SQL-скрипт (в файл `solution.sql`), который выполняет следующие действия:

1. Создайте группу (роль без права логина) с именем `analytics_group`.
2. Создайте пользователя (роль с правом логина) с именем `bi_user` и паролем `bi_pass_123`.
3. Включите пользователя `bi_user` в группу `analytics_group`.
4. Выдайте группе `analytics_group` права на подключение (`CONNECT`) к базе данных `shop_db`.
5. Выдайте группе `analytics_group` права на использование (`USAGE`) схемы `public`.
6. Выдайте группе `analytics_group` права только на чтение (`SELECT`) данных из таблиц `orders` и `products`.

*Подсказка: Выполняйте команды последовательно. Если вы ошиблись, можно использовать `DROP ROLE` или `REVOKE` для отмены действий.*

## 🏆 Задания для самостоятельной работы

Попробуйте выполнить следующие задачи (решения не предоставляются, проверьте себя в консоли psql):

1. **Создание сервисного пользователя:** Создайте пользователя `support_bot`, выдайте ему права на чтение всех таблиц и права на `UPDATE` только для таблицы `users`.
2. **Отзыв прав:** Отзовите права на чтение таблицы `products` у `analytics_group` (попробуйте использовать команду `REVOKE`).
3. **Проверка доступов:** Зайдите в базу данных под созданным пользователем `bi_user` (`psql -U bi_user -d shop_db -h 127.0.0.1 -W`) и попробуйте сделать `SELECT` из таблицы `users`. Вы должны получить ошибку *permission denied*. Затем попробуйте сделать `SELECT` из `orders` — запрос должен выполниться успешно.
