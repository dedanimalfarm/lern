# Модуль 2: Соединение таблиц (JOIN)

## 📖 Теоретическая часть (Что нужно знать)

Перед выполнением практических заданий необходимо самостоятельно изучить следующие темы:
- Понятие первичного ключа (Primary Key) и внешнего ключа (Foreign Key).
- Внутреннее соединение таблиц: `INNER JOIN` (или просто `JOIN`).
- Внешние соединения: `LEFT JOIN`, `RIGHT JOIN`, `FULL OUTER JOIN`.
- Декартово произведение таблиц (`CROSS JOIN`) и почему нужно всегда указывать условие `ON`.
- Использование псевдонимов (Aliases) для столбцов и таблиц (ключевое слово `AS`).

## ❓ Вопросы для самопроверки

1. Что произойдет, если использовать `LEFT JOIN`, но поменять таблицы местами (сначала `orders`, затем `users`)? Чем результат будет отличаться от `INNER JOIN` в данном конкретном случае?
2. В каком случае `RIGHT JOIN` может быть незаменим, или его всегда можно переписать через `LEFT JOIN`?
3. Что будет, если забыть написать условие `ON` в конструкции `JOIN`? 
4. Если в таблице `A` 10 строк, а в таблице `B` 5 строк, и вы делаете `A INNER JOIN B`, какое максимальное количество строк может вернуть запрос? А минимальное?
