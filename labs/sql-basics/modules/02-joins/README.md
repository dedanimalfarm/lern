# Модуль 2: Соединение таблиц (JOIN)

## 📖 Теоретическая часть

В правильно спроектированных (нормализованных) базах данных информация не дублируется, а распределяется по нескольким таблицам. Например, данные о покупателе лежат в таблице `users`, а его покупки — в `orders`. Связь между ними осуществляется через внешний ключ (Foreign Key) — в таблице `orders` хранится `user_id`.

Чтобы собрать эти данные воедино в одном запросе, используются `JOIN`-ы.

### 1. Внутреннее соединение (INNER JOIN)
Возвращает только те строки, для которых нашлось совпадение в ОБЕИХ таблицах.
```sql
SELECT orders.id, orders.order_date, users.name 
FROM orders
INNER JOIN users ON orders.user_id = users.id;
```
*Если в базе есть пользователь без заказов, он не попадет в итоговую выборку.* Слово `INNER` часто опускают и пишут просто `JOIN`.

### 2. Левое внешнее соединение (LEFT JOIN)
Возвращает ВСЕ строки из "левой" (первой указанной) таблицы и только совпавшие строки из "правой". Если для строки из левой таблицы совпадений в правой нет, то поля правой таблицы заполняются значениями `NULL`.
```sql
SELECT users.name, orders.id 
FROM users
LEFT JOIN orders ON users.id = orders.user_id;
```
*Этот запрос выведет всех пользователей. Если пользователь ничего не покупал, в поле `orders.id` будет `NULL`.*

### 3. Псевдонимы (Aliases)
При соединении нескольких таблиц писать их полные названия неудобно. Для этого используют псевдонимы (оператор `AS` или просто пробел):
```sql
SELECT o.id, u.name 
FROM orders o
JOIN users u ON o.user_id = u.id;
```
