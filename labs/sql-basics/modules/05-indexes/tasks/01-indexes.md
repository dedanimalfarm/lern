# Задание 1: Анализ планов запросов и создание индексов

База данных: `shop_db`

## Шаги выполнения

1. **Исследование автоматического индекса:**
   Запустите `EXPLAIN ANALYZE` для запроса, который ищет товар по уникальному имени:
   ```sql
   SELECT * FROM products WHERE name = 'Product_25';
   ```
   Найдите в выводе тип сканирования. Должен быть `Index Scan` (или `Index Only Scan`), так как колонка `name` имеет ограничение `UNIQUE`, для которого СУБД автоматически создает индекс.

2. **Исследование неиндексированной колонки:**
   Запустите `EXPLAIN ANALYZE` для запроса, который фильтрует пользователей по дате регистрации:
   ```sql
   SELECT * FROM users WHERE registration_date = '2026-05-15';
   ```
   Какой тип сканирования используется? Сколько строк отфильтровано и отброшено базой данных (`Rows Removed by Filter`)?

3. **Создание индекса:**
   Создайте обычный индекс на колонку `registration_date` в таблице `users`:
   ```sql
   CREATE INDEX idx_users_reg_date ON users(registration_date);
   ```

4. **Проверка работы индекса (диагностический режим):**
   Так как таблица `users` содержит всего 100 строк, планировщик по-прежнему будет выбирать быстрый последовательный доступ (`Seq Scan`). 
   Отключите последовательное сканирование временно в вашей сессии и запустите `EXPLAIN ANALYZE` для запроса из шага 2 повторно:
   ```sql
   SET enable_seqscan = off;
   EXPLAIN ANALYZE SELECT * FROM users WHERE registration_date = '2026-05-15';
   ```
   Убедитесь, что СУБД переключилась на `Index Scan` с использованием вашего индекса `idx_users_reg_date`.

5. **Очистка стенда:**
   Удалите созданный индекс, чтобы вернуть БД в исходное состояние:
   ```sql
   DROP INDEX idx_users_reg_date;
   ```
