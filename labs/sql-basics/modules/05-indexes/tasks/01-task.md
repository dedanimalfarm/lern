# Задание (продвинутое): Индексы и оптимизация (05-indexes)

> Это задание повышенной сложности. Базовое задание — в файле `01-indexes.md`.

Напишите SQL-запросы в файл `solution.sql` в этой же папке:

1. **Диагностика сложного поиска (BUFFERS):**
   Напишите запрос, который выбирает все товары категории `'Электроника'`, цена которых строго больше `50000`. Оберните его в команду `EXPLAIN (ANALYZE, BUFFERS)`, чтобы увидеть количество затронутых буферов оперативной памяти (shared hits).

2. **Создание составного индекса:**
   Напишите команду создания составного индекса (Composite Index) `idx_products_cat_price` для таблицы `products` по полям `category` и `price`.

3. **Проверка составного индекса:**
   Напишите блок команд, который временно отключает `enable_seqscan`, делает `EXPLAIN (ANALYZE, BUFFERS)` для запроса из шага 1 (чтобы убедиться в использовании составного индекса и сокращении количества буферных чтений), а затем возвращает `enable_seqscan` в исходное состояние (`RESET enable_seqscan;`).

4. **Частичный индекс (Partial Index):**
   Часто аналитикам требуется выгружать отмененные заказы (`status = 'CANCELLED'`).
   Создайте частичный индекс `idx_orders_cancelled` на таблицу `orders` по полю `user_id` только для строк, у которых `status = 'CANCELLED'`.

5. **Покрывающий индекс (Covering Index) и Index Only Scan:**
   Мы часто делаем выборку названий и цен товаров по конкретной категории.
   Создайте покрывающий индекс `idx_products_covering` на таблицу `products` по полю `category`, добавив (покрыв) поля `name` и `price` с помощью секции `INCLUDE`.

6. **Очистка стенда:**
   Напишите команды удаления всех трех созданных в этой практической работе индексов (`idx_products_cat_price`, `idx_orders_cancelled`, `idx_products_covering`), чтобы вернуть базу данных к исходному состоянию.
