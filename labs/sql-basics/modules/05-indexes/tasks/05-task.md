# Задание (продвинутое): Индексы и оптимизация (05-indexes)

> Это задание повышенной сложности. Базовое задание — в файле `05-indexes.md`.

Напишите SQL-запросы в файл `solution.sql` в этой же папке:

1. **Диагностика сложного поиска (BUFFERS):**
   Напишите запрос, который выбирает все фильмы с рейтингом `'R'`, стоимость аренды которых строго больше `2.00`. Оберните его в команду `EXPLAIN (ANALYZE, BUFFERS)`, чтобы увидеть количество затронутых буферов оперативной памяти (shared hits).

2. **Создание составного индекса:**
   Напишите команду создания составного индекса (Composite Index) `idx_film_rating_rate` для таблицы `film` по полям `rating` и `rental_rate`.

3. **Проверка составного индекса:**
   Напишите блок команд, который временно отключает `enable_seqscan`, делает `EXPLAIN (ANALYZE, BUFFERS)` для запроса из шага 1 (чтобы убедиться в использовании составного индекса и сокращении количества буферных чтений), а затем возвращает `enable_seqscan` в исходное состояние (`RESET enable_seqscan;`).

4. **Частичный индекс (Partial Index):**
   Часто аналитикам требуется выгружать невозвращенные аренды (`return_date IS NULL`).
   Создайте частичный индекс `idx_rental_not_returned` на таблицу `rental` по полю `customer_id` только для строк, у которых `return_date IS NULL`.

5. **Покрывающий индекс (Covering Index) и Index Only Scan:**
   Мы часто делаем выборку названий и стоимости аренды фильмов по конкретному рейтингу.
   Создайте покрывающий индекс `idx_film_covering` на таблицу `film` по полю `rating`, добавив (покрыв) поля `title` и `rental_rate` с помощью секции `INCLUDE`.

6. **Очистка стенда:**
   Напишите команды удаления всех трех созданных в этой практической работе индексов (`idx_film_rating_rate`, `idx_rental_not_returned`, `idx_film_covering`), чтобы вернуть базу данных к исходному состоянию.
