# Задание 1: Анализ планов запросов и создание индексов

База данных: `pagila`

## Шаги выполнения

1. **Исследование автоматического индекса:**
   Запустите `EXPLAIN ANALYZE` для запроса, который ищет клиента по уникальному идентификатору:
   ```sql
   SELECT * FROM customer WHERE customer_id = 10;
   ```
   Найдите в выводе тип сканирования. Должен быть `Index Scan` (или `Index Only Scan`), так как колонка `customer_id` является первичным ключом (`PRIMARY KEY`), для которого СУБД автоматически создает индекс.

2. **Исследование неиндексированной колонки:**
   Запустите `EXPLAIN ANALYZE` для запроса, который фильтрует клиентов по дате регистрации:
   ```sql
   SELECT * FROM customer WHERE create_date = '2006-02-14';
   ```
   Какой тип сканирования используется? Сколько строк отфильтровано и отброшено базой данных (`Rows Removed by Filter`)?

3. **Создание индекса:**
   Создайте обычный индекс на колонку `create_date` в таблице `customer`:
   ```sql
   CREATE INDEX idx_customer_create_date ON customer(create_date);
   ```

4. **Проверка работы индекса (диагностический режим):**
   Так как таблица `customer` содержит всего около 600 строк, планировщик по-прежнему может выбирать быстрый последовательный доступ (`Seq Scan`). 
   Отключите последовательное сканирование временно в вашей сессии и запустите `EXPLAIN ANALYZE` для запроса из шага 2 повторно:
   ```sql
   SET enable_seqscan = off;
   EXPLAIN ANALYZE SELECT * FROM customer WHERE create_date = '2006-02-14';
   ```
   Убедитесь, что СУБД переключилась на `Index Scan` с использованием вашего индекса `idx_customer_create_date`.

5. **Очистка стенда:**
   Удалите созданный индекс, чтобы вернуть БД в исходное состояние:
   ```sql
   DROP INDEX idx_customer_create_date;
   ```
