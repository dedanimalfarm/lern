# Задание 1: Создание пользователей и выдача прав

## Легенда
К вам пришла команда разработчиков (developer_team). Им нужна новая база данных `app_db`. Также вам необходимо создать отдельного пользователя `app_user` для подключения приложения с правами на чтение и запись, и пользователя `app_reader` только для чтения (read-only).

## Шаги выполнения

1. **Создание базы данных**:
   - Подключитесь к СУБД от имени суперпользователя `postgres`.
   - Создайте базу данных `app_db`.

2. **Создание пользователей (ролей)**:
   - Создайте роль `app_user` с паролем `app_password` и правом на вход (LOGIN).
   - Создайте роль `app_reader` с паролем `read_password` и правом на вход.

3. **Создание структуры (опционально, для теста)**:
   - Подключитесь к базе `app_db` (`\c app_db`).
   - Создайте тестовую таблицу `users (id serial primary key, username varchar(50))`.
   - Вставьте в нее одну тестовую запись.

4. **Выдача прав**:
   - Дайте пользователю `app_user` полные права (ALL PRIVILEGES) на все таблицы в схеме `public` базы `app_db`. И разрешите использование последовательностей (sequences).
   - Дайте пользователю `app_reader` только право на чтение (SELECT) всех таблиц в схеме `public` базы `app_db`.

5. **Проверка**:
   - Подключитесь к базе `app_db` под пользователем `app_reader`: `psql -U app_reader -d app_db -h 127.0.0.1 -W`.
   - Попробуйте выполнить `SELECT * FROM users;` (должно работать).
   - Попробуйте выполнить `INSERT INTO users (username) VALUES ('test');` (должна быть ошибка доступа `permission denied`).
   - Подключитесь под `app_user` и убедитесь, что `INSERT` работает.
