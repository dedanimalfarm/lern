# Модуль 2: Пользователи, роли и права доступа

## Цель модуля
Научиться создавать базы данных, управлять ролями (пользователями) и настраивать права доступа (привилегии) к таблицам и базам.

## Теория
В PostgreSQL нет жесткого разделения между пользователями и группами. Вместо этого используется концепция "Ролей" (Role). Роль может иметь атрибут `LOGIN` (тогда она выступает как пользователь) или не иметь его (тогда она выступает как группа).

Основные команды:
- `CREATE ROLE name [ [ WITH ] option [ ... ] ]`
- `CREATE USER name` (синоним `CREATE ROLE name WITH LOGIN`)
- `CREATE DATABASE name`
- `GRANT privilege ON object TO role`
- `REVOKE privilege ON object FROM role`

Схемы (Schemas) в PostgreSQL — это пространства имен внутри базы данных. По умолчанию все объекты создаются в схеме `public`.
