# Модуль 3: Резервное копирование и восстановление

## Цель модуля
Изучить логическое резервное копирование базы данных (dump) и её восстановление из резервной копии.

## Теория
В PostgreSQL существует два основных типа бэкапов:
1. **Логический бэкап**: создание SQL-скрипта (или бинарного архива), содержащего структуры и данные. Осуществляется утилитой `pg_dump` (или `pg_dumpall` для всех баз и глобальных объектов типа ролей).
2. **Физический бэкап** (Base backup): копирование файлов с диска (используется утилита `pg_basebackup` для репликации или Point-in-Time Recovery).

В этом модуле мы рассмотрим логический бэкап с использованием утилит:
- `pg_dump` — выгрузка базы. Поддерживает текстовый формат (plain SQL) и сжатый (custom format `-Fc`).
- `pg_restore` — восстановление базы данных из архива custom format.
