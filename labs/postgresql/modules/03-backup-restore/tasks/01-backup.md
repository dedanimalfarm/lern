# Задание 1: Создание дампа и восстановление

## Легенда
Приложение случайно удалило часть важных данных из базы `app_db` (из модуля 2). Хорошо, что вы можете сначала сделать бэкап, "сломать" базу и потом успешно ее восстановить.

## Шаги выполнения

1. **Создание бэкапа**:
   - Используя утилиту `pg_dump`, сделайте дамп базы данных `app_db` в формате *custom* (флаг `-Fc`).
   - Сохраните файл дампа как `/tmp/app_db_backup.dump`.

2. **Имитация аварии**:
   - Подключитесь к базе `app_db` (как `postgres` или `app_user`).
   - Удалите таблицу `users` со всеми данными: `DROP TABLE users;`.
   - Убедитесь, что таблица исчезла.

3. **Восстановление**:
   - Используя утилиту `pg_restore`, восстановите базу данных `app_db` из созданного дампа `/tmp/app_db_backup.dump`.
   - Подсказка: может потребоваться запустить восстановление от имени пользователя `postgres`.

4. **Проверка**:
   - Подключитесь к базе `app_db` и выполните `SELECT * FROM users;`.
   - Данные и структура таблицы должны вернуться в исходное состояние.
