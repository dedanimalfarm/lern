# Модуль 4: Обслуживание базы данных

## Цель модуля
Понять механизмы очистки "мертвых" строк (VACUUM), обновления статистики (ANALYZE) и просмотра активных сессий.

## Теория
Особенность архитектуры PostgreSQL (MVCC - Multi-Version Concurrency Control) заключается в том, что при обновлении (`UPDATE`) или удалении (`DELETE`) строк старые версии данных не удаляются сразу физически с диска. Вместо этого они помечаются как "мертвые" (dead tuples).

- `VACUUM` — процесс очистки мертвых строк, который делает это место доступным для новых вставок.
- `ANALYZE` — процесс сбора статистики по распределению данных в таблицах, который необходим планировщику запросов (Query Planner) для выбора оптимального плана выполнения (индексы или seq scan).
- Автоматически работает демон `autovacuum`, однако администратор должен уметь запускать эти процессы вручную и проверять их.

Системное представление `pg_stat_activity` позволяет увидеть, какие запросы выполняются на сервере прямо сейчас.
