# Задание 1: Ручной Vacuum и мониторинг

## Легенда
Вы заметили, что в одной из баз данных активно происходит вставка и удаление записей. Нужно проверить размер таблицы, выполнить ручную очистку и обновить статистику.

## Шаги выполнения

1. **Генерация нагрузки**:
   - Подключитесь к базе `app_db`.
   - Создайте таблицу для тестов: `CREATE TABLE test_vac (id serial, data text);`
   - Заполните её тестовыми данными: 
     `INSERT INTO test_vac (data) SELECT 'test_string_' || generate_series(1, 100000);`
   - Посмотрите размер таблицы: `SELECT pg_size_pretty(pg_relation_size('test_vac'));`

2. **Создание "мертвых" строк**:
   - Удалите половину записей: `DELETE FROM test_vac WHERE id % 2 = 0;`
   - Снова проверьте размер таблицы (он не должен был уменьшиться, так как мертвые строки всё еще занимают место).

3. **Запуск VACUUM и ANALYZE**:
   - Запустите команду `VACUUM VERBOSE test_vac;`. Изучите вывод.
   - Запустите команду `ANALYZE test_vac;` для обновления статистики планировщика.
   - *Дополнительно*: Выполните `VACUUM FULL test_vac;` и посмотрите размер таблицы снова. (Внимание: `VACUUM FULL` блокирует таблицу, в продакшене используется редко).

4. **Просмотр активных сессий**:
   - Выполните запрос к системному представлению: `SELECT pid, usename, state, query FROM pg_stat_activity;`
   - Найдите свой текущий запрос в выводе.
