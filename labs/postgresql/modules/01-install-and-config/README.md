# Модуль 1: Установка и базовая конфигурация PostgreSQL

## Цель модуля

Научиться устанавливать PostgreSQL на Linux, управлять системным сервисом и выполнять базовую конфигурацию сетевого доступа.

## Теория

PostgreSQL — это мощная объектно-реляционная система управления базами данных (СУБД) с открытым исходным кодом.
При установке на Linux, по умолчанию создается системный пользователь `postgres` и одноименная роль суперпользователя в БД.

### Основные конфигурационные файлы
По умолчанию в Debian/Ubuntu они находятся в `/etc/postgresql/<version>/main/`:
1. `postgresql.conf` — основной файл конфигурации сервера (память, порты, директории). Важный параметр `listen_addresses` определяет, на каких IP-адресах сервер принимает подключения (по умолчанию только `localhost`).
2. `pg_hba.conf` — (Host-Based Authentication) управляет клиентской аутентификацией. В нем настраивается, какие пользователи, с каких IP-адресов и баз данных могут подключаться, а также методы проверки пароля (например, `scram-sha-256`, `md5`, `peer`).
