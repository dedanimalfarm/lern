## Часть 1: Архитектура и установка

### Теория: Как Debian/Ubuntu управляет PostgreSQL
В отличие от многих других дистрибутивов, семейство Debian использует концепцию "кластеров" PostgreSQL (не путать с физическими или логическими кластерами серверов). "Кластер" в терминах PostgreSQL — это набор баз данных, управляемых одним экземпляром сервера. В Ubuntu вы можете установить сразу несколько версий PostgreSQL (например, 12, 14 и 16) и они будут работать параллельно на разных портах.

Конфигурационные файлы отделены от данных:
- Данные лежат в `/var/lib/postgresql/<версия>/main`
- Конфиги лежат в `/etc/postgresql/<версия>/main`
- Бинарники лежат в `/usr/lib/postgresql/<версия>/bin`

### Практика: Установка сервера
Обновите индекс пакетов и установите PostgreSQL и набор дополнительных утилит:

```bash
sudo apt update
sudo apt install -y postgresql postgresql-contrib
```

После установки сервис должен быть запущен автоматически. Проверьте статус:

```bash
systemctl status postgresql
```

Убедитесь, что сервер слушает локальный порт:
```bash
ss -tlnp | grep 5432
```

## Часть 2: Конфигурация postgresql.conf

### Теория: Основной файл конфигурации
Файл `postgresql.conf` управляет поведением сервера. По умолчанию PostgreSQL настроен на прослушивание только локального интерфейса (localhost). Это делается в целях безопасности.

### Практика: Изменение параметров
Откройте файл конфигурации:
```bash
sudo nano /etc/postgresql/14/main/postgresql.conf
```
Найдите директиву `listen_addresses`. Раскомментируйте ее и измените значение:
```ini
listen_addresses = '*'
```
Это укажет серверу принимать подключения на всех доступных сетевых интерфейсах.

Также найдите параметры, отвечающие за ресурсы:
- `max_connections` (по умолчанию обычно 100).
- `shared_buffers` (по умолчанию 128MB). Измените это значение на `256MB`.

Перезапустите сервис, чтобы применить изменения:
```bash
sudo systemctl restart postgresql
```

## Часть 3: Настройка аутентификации (pg_hba.conf)

### Теория: Host-Based Authentication
Файл `pg_hba.conf` контролирует, КТО и ОТКУДА имеет право подключаться к серверу.
Каждая строка описывает правило:
`TYPE  DATABASE  USER  ADDRESS  METHOD`

Популярные методы:
- `peer` - доверяет системному пользователю Linux (только для локальных сокетов).
- `md5` / `scram-sha-256` - требует пароль.
- `reject` - явно отклоняет подключение.
- `trust` - доверяет всем без пароля (опасно, никогда не используйте в production).

### Практика: Разрешение внешних подключений
Добавьте строку в конец `pg_hba.conf`:
```
host    all             all             0.0.0.0/0               scram-sha-256
```

Примените изменения без перезапуска сервера (reload):
```bash
sudo systemctl reload postgresql
```

## Часть 4: Troubleshooting - отказ в обслуживании

В директории модуля (`broken/scenario-01/`) вы найдете файл `pg_hba.conf`.
Скопируйте его поверх вашего текущего конфигурационного файла, выполните `reload` и попробуйте подключиться к СУБД по IP.

Вы получите ошибку `FATAL: no pg_hba.conf entry for host...`.
**Задача:** Изучите содержимое файла `pg_hba.conf`, найдите проблему и исправьте ее. Подсказка: PostgreSQL читает файл сверху вниз и применяет первое совпавшее правило.
