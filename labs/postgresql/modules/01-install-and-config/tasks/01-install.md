# Задание 1: Установка и настройка подключений

## Легенда
Вам выделили чистый сервер для разработки. Ваша задача — установить СУБД PostgreSQL и разрешить к ней доступ по сети с парольной аутентификацией.

## Шаги выполнения

1. **Установка PostgreSQL**:
   - Установите пакет `postgresql` и `postgresql-contrib` используя пакетный менеджер вашей ОС (например, `apt`).
   - Убедитесь, что служба `postgresql` запущена и добавлена в автозагрузку.

2. **Первичный вход**:
   - Переключитесь на пользователя `postgres` в системе.
   - Зайдите в консоль СУБД командой `psql`.
   - Задайте пароль суперпользователю `postgres`: `\password postgres` (например, `secretpassword`).
   - Выйдите из `psql` (`\q`).

3. **Настройка сетевого доступа (`postgresql.conf`)**:
   - Найдите файл `postgresql.conf` (в Ubuntu обычно `/etc/postgresql/<version>/main/postgresql.conf`).
   - Измените параметр `listen_addresses`, чтобы сервер слушал все интерфейсы: `listen_addresses = '*'`.

4. **Настройка аутентификации (`pg_hba.conf`)**:
   - Найдите файл `pg_hba.conf` в той же директории.
   - Добавьте правило, разрешающее подключение всем пользователям ко всем базам с любого IP-адреса по паролю (рекомендуется метод `scram-sha-256` или `md5`).
   - Пример строки: `host all all 0.0.0.0/0 md5`.

5. **Применение изменений**:
   - Перезапустите службу PostgreSQL для применения настроек: `systemctl restart postgresql`.

6. **Проверка**:
   - Попробуйте подключиться к СУБД локально через сетевой интерфейс: `psql -h 127.0.0.1 -U postgres -W`.
   - Если подключение прошло успешно и запрашивается пароль — задание выполнено!
